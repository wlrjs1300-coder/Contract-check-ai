# ADR-012: Orphan Request Directory Cleanup

## 상태
accepted

## 배경
동기식 요청 처리에서 비정상 종료 또는 서버 재시작이 발생하면 임시 요청 디렉터리가 잔존할 수 있다.
잉여 임시 데이터는 개인정보 유추 또는 저장 위반으로 이어질 수 있다.

## 결정
- 각 요청 디렉터리에 민감정보를 제외한 marker를 기록한다.
- startup에서 temp root를 한 번 스캔해 orphan 후보를 정리한다.
- 삭제 경로는 기존 `cleanup_request_directory`의 경계 검증을 공유한다.
- marker 누락/파싱 불가 디렉터리는 즉시 삭제하지 않는다(보수적 보존).
- 재시도 제한 값을 환경변수로 두고, 삭제 실패를 실패 카운트로 집계한다.

## 처리 모드
- 기본 동작은 일회성 분석 모드로 유지한다.
- 보관형 모드는 별도 인증/보관 정책이 활성화되었을 때 확장한다.

## 대안 비교
- 즉시 전체 삭제: 구현은 단순하나 오탐/데이터 손실 위험 증가.
- 무시: 구현 비용이 낮지만 종료 후 잔존 위험이 남음.
- 제안한 startup sweeper: 기존 계약 변경 최소화와 보안 경계 강화의 균형이 가장 우수함.

## 결과
- `feat/v0.7.3-original-temp-cleanup`에서 임시 잔존 관리 체계를 도입.
- 재시작 경계 후 처리 상태를 보수적으로 유지.
