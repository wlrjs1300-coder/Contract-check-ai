# 공개 기록 관리 규칙

## 1. 목적

공개 저장소에는 프로젝트의 최종 코드, 설계 문서, 검증 기록, 버전별 체크리스트만 포함한다.

작업 중간 산출물, 임시 메모, 검토 전 입력/출력 원문, 비공개 검토 자료는 공개 저장소에 포함하지 않는다.
검토 전 입력/출력 원문은 보안 규칙 문서의 정의를 따른다.

공개 기록의 범위에는 코드와 문서뿐 아니라 커밋 메시지, PR 제목/본문, PR 댓글, 이슈, 스크린샷, 터미널 출력도 포함한다.

## 2. 공개 저장소 포함 대상

- 최종 코드
- README
- 버전별 체크리스트
- 기술 의사결정 문서
- API 명세
- DB 설계
- 보안 정책
- 검증 결과
- 정제된 커밋 메시지
- 정제된 PR 제목과 본문
- 정제된 PR 댓글과 이슈
- 민감정보가 제거된 스크린샷과 터미널 출력
- 가상의 샘플 데이터

## 3. 공개 저장소 제외 대상

- 작업 중간 메모
- 검토 전 입력/출력 원문
- 임시 분석 결과
- 원본 계약서 파일
- 개인정보가 포함된 파일
- API Key, Secret, Token
- 로컬 환경 설정 파일
- 로컬 사용자명
- 로컬 파일 경로
- 이메일 주소
- 저장소 인증 정보
- 민감정보가 포함된 화면 캡처
- 민감정보가 포함된 터미널 출력
- 자동 생성 메타데이터
- 자동 삽입된 공동 작성자 문구

로컬 사용자명, 로컬 파일 경로, 이메일 주소, 저장소 인증 정보는 터미널 출력이나 화면 캡처에 노출될 수 있으므로 공개 전에 반드시 확인한다.

## 4. 작성 원칙

문서와 커밋 메시지는 작업 과정이 아니라 최종 결정과 구현 결과를 중심으로 작성한다.

좋은 기록은 다음 질문에 답해야 한다.

- 무엇을 만들었는가
- 왜 그렇게 결정했는가
- 어떤 파일이 바뀌었는가
- 어떻게 검증했는가
- 남은 이슈는 무엇인가

## 5. 사용하지 않을 표현

공개 문서와 커밋 메시지에는 다음과 같은 표현을 사용하지 않는다.

```text
검토 전 제안이다
검토 전 생성된 내용이다
입력 원문
출력 원문
생성 결과 반영
자동 생성됨
```

## 6. 사용할 표현

다음과 같이 기술적 판단과 구현 결과 중심으로 작성한다.

```text
개인정보 보호를 위해 분석 전 마스킹 단계를 추가했다.
원본 계약서 유출 위험을 줄이기 위해 분석 완료 후 원본 파일 삭제 정책을 적용했다.
문서 분석 파이프라인의 책임 분리를 위해 업로드, 텍스트 추출, 마스킹, 분석 단계를 분리했다.
계약서 원문이 로그에 남지 않도록 로그 보안 정책을 정의했다.
```

## 7. 검토 요약 작성 규칙

공개 체크리스트에는 원문 기록 대신 검토 후 확정한 요약만 남긴다.

좋은 예:

```text
근로계약서와 주택 임대차계약서를 MVP 대상으로 선정했다.
개인정보 보호 부담을 고려해 원본 파일 저장을 제외했다.
분석 결과는 법률 자문이 아닌 참고용 사전점검 결과로 제공한다.
```

나쁜 예:

```text
검토 전 응답을 그대로 붙여넣음
요청 원문과 응답 원문을 그대로 기록
```

## 8. 최종 검증 기록과 원문 기록 구분

공개 가능한 기록은 검토 후 정리된 최종 판단과 구현 결과이다.

공개 가능한 최종 검증 기록:

- 변경 파일 목록
- 검증한 명령어 이름
- 검증 결과 요약
- 남은 이슈 요약
- 다음 작업 기준

공개하면 안 되는 원문 기록:

- 실제 사용자 요청/응답 원문
- 계약서 원문
- 마스킹 전 텍스트
- 터미널 전체 출력 중 로컬 경로, 사용자명, 이메일 주소, 인증 정보가 포함된 내용
- 민감정보가 보이는 스크린샷
- 비공개 검토 자료 전문

## 9. 커밋 전 확인

```bash
git status
git diff
git diff --cached
git diff --cached --name-only
git log -1 --pretty=full
```

확인할 항목:

- 공개 저장소 제외 대상이 포함되지 않았는가
- 자동 삽입된 공동 작성자 문구가 없는가
- 작업 중간 산출물이 포함되지 않았는가
- 최종 판단과 구현 결과 중심으로 작성되었는가
- 커밋 메시지, PR 제목/본문, PR 댓글, 이슈에 민감정보가 포함되지 않았는가
- 스크린샷과 터미널 출력에 로컬 사용자명, 로컬 파일 경로, 이메일 주소, 저장소 인증 정보가 포함되지 않았는가
