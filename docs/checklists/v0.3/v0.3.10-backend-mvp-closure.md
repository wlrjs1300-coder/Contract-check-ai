## 2. 작업 목적

v0.3.0부터 v0.3.9까지 구현된 백엔드 MVP 흐름을 통합 검증하고, 실제 병합 이력과 테스트 결과를 기준으로 v0.3.x 단계를 종료한다.

이번 버전은 새로운 기능 추가가 아니라 기존 구현의 범위, 완료 상태, 알려진 한계, 후속 작업 진입 조건을 정리하는 종료 단계다.

- Provider 실행 정책

## 7. 통합 검증 기준

- 전체 pytest 통과
- 전체 Ruff 통과
- 작업 트리 clean
- 버전별 merge commit 확인
- 예상 밖 파일 변경 없음
- 실제 계약서·실제 개인정보·API Key 미포함
- 외부 AI 호출 없음
- 동결 fixture·expected 변경 없음
- 실패 결과 미저장
- 부분 결과 rollback 유지
- 개인정보 재생성 결과 차단
- 출력 안전성 실패 결과 차단

## 8. 알려진 한계

- 실제 외부 AI Provider가 연결되지 않음
- 실제 Provider별 예외 adapter가 없음
- 운영 timeout·retry·backoff 값 미확정
- 실패 사유 DB 저장 미구현
- 실제 계약서 recall·precision 미검증
- OCR·PDF 미지원
- Reviewer 워크플로 미구현
- 검토 대기 상태 미구현
- 인증·인가 미구현
- 프론트엔드 미연동
- MySQL 미전환
- 배포·CI 미구성
- 작업 프로세스 재시작 복구 미지원

## 9. 후속 단계 진입 조건

v0.3.x 종료 후 다음 단계에서는 아래 중 우선순위를 정해 진행한다.

- 실제 Provider 연결 승인 및 adapter 구현
- 실패 사유 DB 저장
- 프론트엔드 API 연동
- 인증·인가
- MySQL 전환
- 배포·CI
- 비동기 작업 큐와 작업 복구
- 결과 스키마 확장
- Reviewer 워크플로

실제 외부 Provider 연결은 공급자 데이터 정책, 비용, endpoint, model, timeout, retry, backoff, 실제 전송 데이터 범위에 대한 별도 승인 후 진행한다.
