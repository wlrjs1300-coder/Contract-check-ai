## 2. 작업 목적

provider가 반환한 분석 결과의 `reference_id`가 현재 처리 중인 조항의 `reference_id`와 정확히 일치하는지 검증한다.

잘못된 provider 응답이 다른 조항이나 존재하지 않는 조항에 연결되어 저장되는 것을 차단하고, 실패 시 기존 rollback 및 `failed` 상태 보존 규칙을 유지한다.

## 5. 설계 원칙

- 조항 자체의 `reference_id` 검증과 provider 결과의 `reference_id` 검증을 분리한다.
- provider 결과는 현재 처리 중인 조항과 정확히 1:1로 연결돼야 한다.
- 검증 실패 시 결과를 저장하지 않는다.
- 검증 실패 시 기존 트랜잭션 rollback 규칙을 유지한다.
- API 계층은 변경하지 않는다.
