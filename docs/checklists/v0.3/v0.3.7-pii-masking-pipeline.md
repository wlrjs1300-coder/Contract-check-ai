## 2. 작업 목적

v0.2.1 PR-3에서 승인·동결된 합성 개인정보 fixture와 expected 기준을 백엔드 분석 파이프라인에 통합한다.

원본 Clause 객체 전체를 Provider에 전달하지 않고, 개인정보 탐지·마스킹과 residual 검증을 통과한 조항 단위 입력만 Provider에 전달한다.

## 5. 설계 원칙

- 동결 fixture와 expected 파일을 수정하지 않는다.
- 개인정보 탐지 결과에 원문 값 필드를 포함하지 않는다.
- 원본 Clause.body를 수정하지 않는다.
- Provider에는 reference_id와 마스킹된 텍스트 등 허용된 최소 필드만 전달한다.
- residual 개인정보가 탐지되면 Provider를 호출하지 않는다.
- 마스킹 실패는 분석 파이프라인 실패로 처리한다.
- 계약서 본문과 개인정보를 로그에 기록하지 않는다.
- 실제 외부 전송은 별도 승인 전까지 수행하지 않는다.
- 기존 분석 결과 검증과 rollback 규칙을 유지한다.
