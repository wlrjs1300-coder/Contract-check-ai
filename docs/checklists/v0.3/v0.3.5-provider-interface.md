## 2. 작업 목적

현재 분석 파이프라인 내부에서 직접 생성하는 합성 분석 결과를 교체 가능한 provider 경계 뒤로 분리한다.

외부 AI API를 연결하지 않고도 provider 호출, 결과 검증, 저장 및 실패 처리 흐름을 테스트할 수 있는 구조를 만든다.

---

## 5. 설계 원칙

- API 계층은 구체적인 provider 구현을 알지 않는다.
- provider는 `AnalysisResultData`를 반환한다.
- 결과 검증과 DB 저장 책임은 분석 파이프라인에 유지한다.
- 기본 실행은 합성 provider를 사용한다.
- 테스트에서는 provider를 교체할 수 있어야 한다.
- provider 오류가 발생하면 기존 rollback 및 `failed` 상태 보존 규칙을 유지한다.
