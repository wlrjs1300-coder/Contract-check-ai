## 2. 작업 목적

Provider 호출 실패를 안전하게 분류하고, 재시도 가능한 오류와 즉시 실패해야 하는 오류를 구분하는 실행 경계를 구현한다.

실제 외부 AI 공급자와 운영 환경이 아직 확정되지 않았으므로 timeout 값, 최대 재시도 횟수, backoff 수치를 전역 운영값으로 확정하지 않는다. 대신 정책 객체와 지연 함수 주입 구조를 제공하고, 합성 Provider를 사용해 동작을 검증한다.

## 5. 설계 원칙

- Provider 예외를 그대로 외부로 노출하지 않는다.
- 실패 사유는 안전한 reason code로 분류한다.
- 재시도 가능 여부는 오류 타입에 명시한다.
- 기본 정책은 외부 호출과 실제 대기 없이 안전하게 동작해야 한다.
- 정책 값은 호출부 또는 테스트에서 명시적으로 주입 가능해야 한다.
- 즉시 실패 오류는 재시도하지 않는다.
- 재시도 가능한 오류도 최대 시도 횟수를 넘지 않는다.
- 실패한 Provider 응답이나 원시 예외를 저장하지 않는다.
- 기존 schema·reference_id·PII·출력 안전성 검증 순서를 유지한다.
- 기존 rollback 및 AnalysisJob failed 처리 규칙을 유지한다.

## 6. 실패 사유 코드 후보

- `PROVIDER_TIMEOUT`
- `PROVIDER_TRANSPORT_ERROR`
- `PROVIDER_RATE_LIMITED`
- `PROVIDER_AUTHENTICATION_ERROR`
- `PROVIDER_NOT_APPROVED`
- `PROVIDER_EXECUTION_FAILED`

이 코드는 실제 외부 Provider 실행 기록이 아니라 합성 실패 시나리오의 안전한 분류 기준이다.
