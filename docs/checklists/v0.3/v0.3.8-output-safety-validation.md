## 2. 작업 목적

v0.2.1 PR-5에서 승인·동결된 출력 안전성 기준을 백엔드 분석 파이프라인에 통합한다.

Provider가 반환한 `AnalysisResultData.summary`를 신뢰하지 않고, 결과 저장 전에 법률 확정 표현, 결과 보장, 직접 법적 행동 지시, 전문가 검토 불필요 표현 및 개인정보 재생성 여부를 검증한다.

현재 백엔드에는 검토 대기 상태와 Reviewer 워크플로가 없으므로 `ALLOW`만 저장을 허용하고, `BLOCK`과 `REVIEW`는 모두 fail-closed 방식으로 차단한다.

## 5. 설계 원칙

- Provider 응답은 항상 신뢰하지 않는 입력으로 취급한다.
- 출력 안전성 검증 전에는 결과를 저장하지 않는다.
- 현재 백엔드에서는 `ALLOW`만 저장을 허용한다.
- `BLOCK`과 `REVIEW`는 모두 결과 저장을 차단한다.
- 개인정보 재생성이 탐지되면 결과 저장을 차단한다.
- 원시 Provider 응답은 저장하거나 로그에 기록하지 않는다.
- 출력 안전성 검사 결과에는 원문 summary를 복제하지 않는다.
- 기존 schema 검증과 reference_id 검증을 유지한다.
- 기존 rollback 및 AnalysisJob failed 처리 규칙을 유지한다.
- 동결 fixture와 expected 파일을 수정하지 않는다.
