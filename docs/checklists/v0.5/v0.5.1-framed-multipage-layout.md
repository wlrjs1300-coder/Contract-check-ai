## 상태와 페이지 전환 규칙

- `ScaffoldPage`가 기존 업로드·문서·분석 작업·결과 상태와 API 핸들러를 계속 소유한다.
- 별도 라우터 없이 `AppPage` UI 상태로 세 페이지를 조건부 렌더링한다.
- 상단 링크는 접근 가능한 URL 형태를 유지하되 클릭 시 전체 새로고침 없이 내부 상태를 전환한다.
- 새로고침 시 안전하게 메인 페이지로 진입한다.
- 메뉴 전환 중 업로드 문서와 분석 상태는 유지한다.
- 새 파일 선택 시 기존 분석 작업과 결과를 초기화하고 문서 분석 페이지를 유지한다.
- 결과 연결 검증이 성공한 경우에만 분석 결과 페이지로 이동한다.

## 조항 페이징 규칙

- 결과 첫 진입 시 첫 조항을 선택한다.
- 조항 목록 버튼으로 직접 선택할 수 있다.
- 이전·다음 버튼과 현재 번호/전체 개수를 제공한다.
- 첫 조항의 이전 버튼과 마지막 조항의 다음 버튼을 비활성화한다.
- 현재 조항은 텍스트, 테두리와 `aria-current`로 구분한다.
- 새 문서 선택으로 결과 화면이 해제되면 다음 결과 진입 시 첫 조항부터 시작한다.

## 반응형

- 320px 이상에서 고정 너비 없이 프레임 여백과 내부 padding을 축소한다.
- 상단 메뉴는 작은 화면에서도 3개 프로그램 페이지 메뉴를 유지한다.
- 홈 2열은 1열, 이용 절차 4열은 2열과 1열로 전환한다.
- 결과 2열은 조항 목록 위·상세 아래의 1열로 전환한다.
- 긴 파일명·제목·본문은 컨테이너 안에서 줄바꿈한다.

## 알려진 한계

- URL은 링크 의미를 제공하지만 새로고침 시 메인으로 안전 진입하며 브라우저 history와 동기화하지 않는다.
- 실제 screen reader와 전문 색상 대비 도구 검증은 별도 수동 확인이 필요하다.
- 320·375·576·768·1024·1440px 실제 브라우저 최종 검증이 필요하다.
- 현재 결과는 합성 Provider 기반이며 실제 AI 품질이나 법률적 적합성 검증이 아니다.
