## 결과

### 변경 파일

- `docs/planning/user-scenarios.md`
- `docs/checklists/v0.1/v0.1.4-user-scenarios.md`
- `docs/05-folder-structure.md`

### 확정 내용

- 비회원은 업로드, 분석, 결과 조회를 할 수 없다.
- 인증된 일반 사용자만 계약서 업로드와 본인 결과 조회를 할 수 있다.
- 분석 중 취소 시나리오는 포함하되 구현 방식은 후속 설계에서 확정한다.
- 실패한 분석은 최소 정보와 실패 상태를 이력에 표시한다.
- 동일 파일 재업로드를 허용한다.
- 계약서 유형 불일치 시 자동 전환하지 않고 사용자 재선택을 요구한다.
- 관리자 기능은 MVP에서 제외한다.
- 전체 서비스 지원 계획은 근로계약서와 주택 임대차계약서를 유지한다.
- v1.0 최초 구현과 기술 검증 우선 대상은 근로계약서 1종으로 둔다.
- 주택 임대차계약서는 후속 구현 대상으로 둔다.

### 검증 결과

- 독립 검토를 완료하고 필수 수정 1건과 권장 수정 3건을 반영했다.
- 개인정보, Secret, 원본 계약서, 검토 전 입력/출력 원문은 포함하지 않았다.
- 실제 계약서 예시와 실제 개인정보는 포함하지 않았다.
- 특정 작업 도구명은 포함하지 않았다.
- 법률 자문, 적법성, 위법성, 무효, 법적 효력, 승소, 반환, 해고 결과를 확정하는 표현은 사용하지 않았다.
- 화면, API, DB, AI 세부 설계는 후속 단계로 분리했다.
- `.agents/` 구조 불일치를 현재 실제 구조에서 보완했다.
- 변경 파일이 예정된 3개 파일로 제한되어 있음을 확인했다.
- `git diff --check`에서 실제 오류가 없음을 확인했다.
- 네트워크 업로드 중단 시나리오를 추가했다.

### 남은 이슈

- §15는 사용자 시나리오 자체가 아니라 구현 우선순위에 관한 내용이므로 후속 버전 계획 정리나 ADR 작성 시 이관 여부를 검토한다.
- v0.1.5에서 책임 범위, 고지 문구, 동의 및 면책 정책을 구체화한다.
- v0.2에서 개인정보 탐지, 마스킹, 원본 삭제 실패 정책을 구체화한다.
- v0.3에서 화면별 사용자 안내 위치와 표현을 구체화한다.
- v0.4에서 API, DB, 권한 검증 방식을 설계한다.
- v0.5에서 계약서 유형 판별, 조항 분석 규칙, AI 프롬프트와 응답 구조를 설계한다.
