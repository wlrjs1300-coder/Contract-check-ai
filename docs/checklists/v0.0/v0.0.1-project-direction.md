## 결과

### 변경 파일

- `docs/checklists/v0.0/v0.0.1-project-direction.md`

### 확정 내용

- 프로젝트명: ContractCheck AI
- 부제: 개인정보 보호형 계약서 사전점검 서비스
- 서비스 한 줄 정의: 개인정보 보호를 고려한 AI 기반 계약서 사전점검 서비스
- 프로젝트 목적:
  - 사용자가 계약서를 업로드하기 전에 주요 위험 조항을 사전에 점검할 수 있도록 돕는 포트폴리오용 MVP 서비스를 구축한다.
  - 법률 자문 서비스가 아니라, 계약서 내용을 이해하기 쉽게 정리하고 위험 가능성이 있는 조항을 안내하는 사전점검 서비스로 정의한다.
- MVP 지원 계약서:
  - 근로계약서
  - 주택 임대차계약서
- MVP 핵심 기능:
  - 계약서 업로드
  - 텍스트 추출
  - 조항 분리
  - 위험 조항 분석
  - 위험도 표시
  - 사용자 친화적인 분석 결과 제공
  - 개인정보 보호를 고려한 업로드/저장/마스킹 기준 적용
- 이번 단계에서 제외할 기능:
  - 결제
  - OCR
  - 관리자 페이지
  - 법률 자문 기능
  - 계약서 자동 작성
  - 전자서명
  - 복잡한 ERP 기능
  - 실사용 법률 검토 대체 기능
- 기술스택 방향:
  - Frontend: React + Vite
  - UI: Bootstrap 5 + Custom CSS
  - Backend: FastAPI
  - Database: MySQL
  - Auth: JWT
  - AI: 외부 AI API 연동 구조
  - Docs: Markdown
  - GitHub: branch / commit / PR / tag 규칙 기반 관리
- 다음 단계 이동 가능 여부: v0.0.1 완료 후 v0.0.2 저장소/Git 운영 규칙 정리 단계로 이동 가능

### 검증 결과

- 프로젝트명, 서비스 정의, 목적, MVP 대상 계약서, 핵심 기능, 제외 범위, 기술스택 방향을 명시했다.
- 법률 자문이나 실사용 법률 검토 대체로 오해될 수 있는 표현을 제외하고 사전점검 서비스로 정리했다.
- 작업 중간 산출물, 민감정보, 원본 계약서 파일 정보는 포함하지 않았다.
- 현재 문서는 공개 GitHub 저장소에 남겨도 되는 정제된 프로젝트 방향성 기록이다.

### 남은 이슈

- 세부 기술스택은 후속 v0.0 작업에서 확정한다.
- Git 운영 규칙은 v0.0.2에서 정리했다.
- 보안/공개 기록 기준은 v0.0.3에서 정리했다.
- `main` 병합과 태그 생성은 아직 진행하지 않았다.
