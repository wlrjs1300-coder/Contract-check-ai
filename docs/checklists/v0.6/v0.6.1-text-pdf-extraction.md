## 목적

기존 TXT 업로드 계약을 유지하면서 텍스트 레이어 PDF를 안전하게 검증·직접 추출하고, 페이지별 결과를 사용자 검토 필요 상태의 별도 extraction resource로 제공한다.

## 알려진 한계

- parser process 격리와 강제 timeout 미구현
- 비정상 종료 orphan sweeper 미구현
- 표준 라이브러리의 경로 검사와 재귀 삭제 사이 TOCTOU 위험이 남아 있으며 OS-level sandbox를 구현하지 않음
- bbox·confidence·표 구조와 시각적 읽기 순서 미지원
- 실제 다양한 PDF font·layout corpus를 사용하지 않음
- 직접 추출 품질 heuristic은 OCR 필요 후보 신호이며 정확성 판정이 아님
- 인증·소유권이 없어 실제 문서를 입력하면 안 됨
- 추출 텍스트 보존 기간과 사용자 확인 후 삭제 전환은 Phase 5에서 결정
