## 목적

PDF·JPG·JPEG·PNG를 P0 입력으로, TXT를 테스트·보조 입력으로 유지하기 전에 지원 계약, OCR 선택 기준, 원본 보호·삭제, 추출 데이터, 오류, 위협과 Phase 2~8 구현 순서를 확정 가능한 초안으로 만든다.

이번 단계는 문서 전용이다. PDF 추출, OCR, API·DB·frontend 동작과 dependency는 구현하지 않는다.

## 현재 코드와의 차이

- 현재 backend는 `.txt` 파일명만 허용하고 1 MiB 전체 바이트를 메모리에서 읽는다.
- 현재는 strict UTF-8 디코딩 뒤 같은 요청에서 조항 분리·SQLite 저장까지 끝나 `processed`를 반환한다.
- 현재 파일 시그니처, PDF parser, image decoder, OCR, 임시 원본 파일과 사용자 확인 revision이 없다.
- 현재 frontend의 accept, 사전 검증, 문구와 type은 TXT 1 MiB 응답에 맞춰져 있다.
- 현재 인증·인가와 사용자별 문서 격리가 없고 실제 분석은 합성 Provider를 사용한다.
- `docs/09-tech-stack.md`의 장기 MySQL·JWT·외부 Provider 방향과 현재 SQLite·무인증 구현은 서로 다른 단계다.

이 차이는 Phase 2 이후 별도 구현·migration·회귀 테스트로 해소하며 이번 문서 작업에서 코드를 변경하지 않는다.
