## 인증 계약

- 공개 경로는 `/health`, OpenAPI 문서, `/auth/register`, `/auth/login`이다.
- 보호 경로는 `Authorization: Bearer` 토큰으로 현재 사용자를 확인한다.
- 인증 실패, 비활성 사용자, `auth_version` 불일치에는 `401`을 반환한다.
- 타 사용자 리소스 접근에는 리소스 존재 여부를 노출하지 않는 고정 `404`를 반환한다.

## JWT 검증

- `JWT_SECRET`은 UTF-8 기준 최소 32바이트여야 한다.
- 토큰 TTL은 5분 이상 60분 이하이며 기본값은 15분이다.
- 필수 claim은 `sub`, `iat`, `exp`, `jti`, `iss`, `aud`, `ver`이다.
- `sub`와 `jti`는 canonical lowercase UUID 문자열만 허용한다.
- `iat`, `exp`, `ver`는 정확한 `int` 타입만 허용하며 `bool`, `float`, 숫자 문자열을 변환하지 않는다.
- `exp > iat`, 미래 `iat` 제한, issuer, audience, signature를 검증한다.
- 토큰 payload에는 email, `owner_id`, resource ID를 넣지 않는다.

## 비밀번호 처리

- 비밀번호는 pwdlib Argon2로 해시하며 평문을 저장하지 않는다.
- 길이는 12자 이상 128자 이하이며 null byte를 거부한다.
- 앞뒤 공백을 임의로 제거하지 않는다.
- 존재하지 않는 email도 dummy Argon2 verify 경로를 실행한다.
- 로그인 실패 응답은 계정 존재 여부와 무관하게 동일하다.

## migration 계약

- revision 체인은 `0001_current_schema_baseline` → `0002_auth_ownership`이다.
- `0002_auth_ownership`은 빈 개발 DB에서만 진행한다.
- 실행 전 `documents` 또는 `extractions`에 row가 하나라도 있으면 즉시 중단한다.
- legacy row를 임의 사용자에게 자동 배정하지 않는다.
- 중단 시 revision은 `0001_current_schema_baseline`에 머물고 기존 row를 유지하며 부분 DDL을 남기지 않는다.
