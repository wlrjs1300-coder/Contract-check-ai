# v0.7.2 계약 데이터 생명주기 및 보안 경계 설계

## 기본 정보
- 버전: v0.7.2
- 브랜치: feat/v0.7.2-data-lifecycle-security
- 작업명: 계약서 데이터 생명주기 및 보안 경계 설계
- 시작일: 2026-07-22
- 완료일:
- 현재 단계: 진행 중

## 목표
- 계약서 입력부터 분석 결과 저장까지의 데이터 이동과 보관 위치를 동일한 경계로 통일한다.
- 원본 파일, 추출 텍스트, 마스킹 데이터, 분석 결과를 구분해 처리한다.
- 보안 게이트를 통해 데이터 보존, 전송, 삭제를 통제한다.
- 보관형 모드와 일회성 모드의 운영 차이를 문서화해 차후 구현 범위를 명확히 한다.

## 필수 확인 항목
- 원본 파일은 분석 종료 후 기본 삭제 대상인지 확인한다.
- 추출 텍스트는 분석 후 기본 보존 정책을 따른다.
- Provider 호출 시 마스킹 전송을 적용한다.
- 삭제 실패 시 경고 및 재시도/운영 개입 절차가 존재한다.
- 실패 시 상태를 `failed`로 전이하고 분석 결과의 보안 상태를 별도 플래그로 관리한다.
- 외부 Provider 의존 시 fail-closed를 적용한다.

## 현재 구현과 목표 상태
- 현재 구현
  - 분석 요청, 업로드, 추출/조항 분리, 분석 결과 반환이 가능함.
  - 일부 경로는 fail-closed 정책이 적용됨.
  - 보안 게이트의 모든 항목이 운영 기본값으로 완전 수렴되지는 않음.
- 목표 상태
  - 처리 모드별로 데이터 생명주기를 엄격히 분리하고 삭제/복구/보관 상태를 추적한다.
  - 원본·중간 산출물은 운영 모드에서만 제한적 허용.
  - 로그와 감사 기록은 최소 수집, 민감영역 분리 저장, 보존 종료 시 정리 보장.

## 처리 모드
- 일회성 안전 분석 모드
  - 기본 모드.
  - 업로드 원본과 임시 추출 산출물을 분석 종료 후 원칙적으로 정리한다.
  - 분석 결과는 최소 단위로 저장하고 raw 원문은 기본 비보관.
- 보관형 계약 관리 모드
  - 인증, 소유권, 접근 통제, 암호화가 갖춰진 조건에서만 활성화.
  - 원본·중간 데이터 보관이 허용될 수 있으며 보관기간·접근권한을 별도 관리한다.
  - 운영 승인 없이 기본 모드로 자동 전환하지 않는다.

## 데이터 생명주기
- 수집: 업로드(원본), 파일명/형식/크기/검증 결과 메타만 초기 기록
- 분석 준비: 임시 디렉터리 생성, 추출 원문 생성, 조항 단위 분할
- 분석 실행: 조항 원문/마스킹 조항 기반 분석, 결과 생성
- 정합성 검증: 실패 경로, 타임아웃, cleanup 동작 점검
- 종료/보관: 모드별 삭제/보관 적용, 감사 로그 기록

## 보안 게이트
- 업로드 게이트
  - MIME/확장자/크기/인코딩/오염 파일 여부 검사.
  - 실패 시 원본 임시 데이터는 즉시 정리한다.
- 분석 게이트
  - 토큰화 후 PII 탐지 결과를 점검하고, 민감도 높은 범위는 fail-closed.
  - Provider 호출은 마스킹/최소 데이터로 제한.
- 저장 게이트
  - DB와 파일 저장 위치를 분리하고, 민감 데이터 저장 여부를 정책으로 판단.
- 삭제 게이트
  - 경보 발생 시 사용자/운영 삭제 우선순위 적용.
  - 삭제 실패는 실패 코드로 남기고 재확인 대상으로 분리한다.

## 범위 외
- 실제 과금/청구 연동은 본 설계 범위에 포함하지 않는다.
- 다국어 UI 번역 품질 향상은 별도 문서에서 관리한다.
- 임시 파일 스윕러의 정교한 운영 자동화는 추후 버전에서 확정한다.

## 검증 기준
- 각 처리 단계에서 민감 데이터 저장 위치가 정책에 정의된 위치만 사용되는지 점검한다.
- 삭제 실패가 발생하면 사용자 분석 가능 상태가 false로 유지되는지 검증한다.
- 외부 Provider 호출 전 최소 데이터 전송이 지켜지는지 확인한다.
- 운영 로그에 원본 파일 경로가 평문으로 남지 않아야 한다.
- 정책 외 항목(삭제 불가 항목·장기 보관 항목)을 명시적으로 문서화한다.
