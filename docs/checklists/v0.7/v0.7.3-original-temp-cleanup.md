## 핵심 보안 결정
- marker는 원본 파일명, 경로, raw 텍스트를 포함하지 않는다.
- marker는 `schema_version`, `request_id`, `created_at_utc`, `lease_updated_at_utc`,
  `process_id`, `cleanup_state`만 기록한다.
- 임시 경로는 요청 디렉터리 `request-*` 후보만 스캔한다.
- `lease_updated_at_utc`가 TTL 경과한 디렉터리만 orphan 후보로 판단한다.
- marker가 없거나 파싱 불가한 디렉터리는 즉시 삭제하지 않고 보수적으로 건너뛴다.
- 삭제는 `cleanup_request_directory` 기존 경계 검증(경로/링크/identity)을 그대로 사용한다.
- 삭제 실패는 실패 카운트를 남기고 개별 요청 처리를 중단하지 않는다.

## 결과 구조
- `scanned_count`

- `removed_count`
- `skipped_active_count`
- `skipped_unsafe_count`

- `status_codes`(경로·예외 미포함 집계)

## 운영 파라미터
- `EXTRACTION_ORPHAN_TTL_SECONDS` (기본값: 3600, 60~86400 범위)
- `EXTRACTION_ORPHAN_CLEANUP_RETRY_COUNT` (기본값: 3, 1~24 범위)
- `EXTRACTION_ORPHAN_CLEANUP_RETRY_DELAY_SECONDS` (기본값: 1.0, 0~10 범위)
