## 인증 주체와 소유권 경계

- 인증 주체는 `User`이다.
- `owner_id`를 직접 저장하는 소유 리소스는 `Document`, `Extraction`이다.
- `AnalysisJob`은 `Document.owner_id`를 따른다.
- `Clause`는 `Document.owner_id`를 따른다.
- `AnalysisResultItem`은 `AnalysisJob`을 거쳐 `Document.owner_id`를 따른다.
- `ExtractionPage`는 `Extraction.owner_id`를 따른다.
- 하위 리소스의 AAD에는 검증된 부모 소유 리소스에서 얻은 `owner_id`를 사용한다.

## Envelope와 AAD

```json
{
  "enc_version": 1,
  "key_id": "synthetic-key-v1",
  "algorithm": "AES-256-GCM",
  "nonce": "...",
  "ciphertext": "..."
}
```

- `nonce`는 Base64로 인코딩한다.
- `ciphertext`는 AESGCM이 반환한 ciphertext와 authentication tag의 결합 payload 전체를 Base64로 인코딩한다.
- plaintext AAD는 envelope에 저장하지 않는다.
- `algorithm`, `enc_version`, `key_id`는 엄격한 타입·allowlist 검증을 거친다.
- envelope의 누락 필드와 추가 필드는 기본적으로 fail-closed 처리한다.
- AAD는 서버가 신뢰 가능한 DB 관계와 호출 context에서 deterministic canonical encoding으로 재생성한다.
- 권장 구성은 `resource_type`, `record_id`, `field_name`, `owner_id`, `schema_version`이다.
- canonical JSON은 UTF-8, key 정렬, 불필요한 공백 제거, 타입 변환 금지, 누락 값 금지 규칙을 따른다.

## `extra_data` 구현 기준

v0.7.5 구현에서는 모델별·schema별 allowlist와 validator를 도입한다. 현재 임의 key 금지는 구현 완료 상태가 아니다.

- 평문 운영 메타데이터: 상태 전이, 동시성, checksum, 관찰성, 생명주기 관리에 필요한 version, status, count, boolean, hash, code
- 암호화 대상 민감 하위 필드: snapshot, reviewed/final text, blocks text, 분석 자연어, evidence excerpt, raw/source fact text, 검토 요약과 실행 제안
- 저장 금지: raw Provider payload, raw token mapping, 마스킹 전 전체 복사본, raw PII, Secret, token, Authorization header, 내부 경로, 임시 이미지, 중복 원문, 디버깅 payload

전체 `extra_data`를 투명하게 한 번에 암호화하지 않는다.

## 테스트 계획

### Keyring

- 누락·malformed JSON·중복 `key_id`·active key 불일치
- malformed Base64·31-byte·33-byte key·unsupported status
- decrypt-only key의 신규 encrypt 거부와 retired key 실패
- raw key의 app state 미저장

### Envelope와 암복호화

- round trip과 nonce에 따른 ciphertext 차이
- ciphertext/authentication tag·nonce tamper
- unknown `key_id`, unsupported version, algorithm mismatch
- 필드 누락·추가 필드·malformed Base64
- 한글 Unicode, 빈 문자열, 대용량 텍스트, null byte 정책
- double encryption 방지

### AAD와 저장 경계

- 잘못된 resource, record, field, owner, schema version 거부
- 외부 입력 AAD를 신뢰하지 않고 서버 재구성값 사용
- 다른 row로 ciphertext 복사 시 복호화 실패
- A등급 필드와 민감 JSON 하위 필드의 raw DB 평문 검색 실패
- ORM/API 정상 복호화와 production 평문 fallback 거부
- 평문과 암호문 동시 저장 거부

### Rotation과 비노출

- active key 신규 쓰기, decrypt-only 기존 읽기와 신규 쓰기 거부
- 새 active key 전환, 점진 재암호화, idempotency, retired key 실패
- 예외·응답·로그에 raw key, 평문, ciphertext, nonce, 환경변수 원문 미노출
- cross-user API `404` 유지
