## 검토

- [ ] 공개 저장소에 올려도 되는 내용인지 확인
- [ ] API Key, 토큰, 계정 정보 등 민감정보가 포함되지 않았는지 확인
- [ ] 원본 계약서 파일이 포함되지 않았는지 확인
- [ ] 작업 중간 산출물 또는 검토 전 입력/출력 원문이 포함되지 않았는지 확인
- [ ] 자동 삽입된 공동 작성자 문구가 포함되지 않았는지 확인

### 문서 작업 검토

- [ ] “법률 자문”으로 오해될 표현이 없는지 확인

### 기능 작업 검토

- [ ] 데이터 없음 상태 확인
- [ ] 보안 관련 로그 확인

## GitHub 반영 상태

체크리스트 파일은 PR 생성 전 검증 상태를 기준으로 작성한다.

PR 생성 이후의 실제 병합 여부는 GitHub PR 기록을 최종 근거로 한다.
필요한 경우 아래 항목에 PR 번호, 대상 브랜치, 병합 확인 방법을 기록한다.

- PR 번호:
- 대상 브랜치:
- 병합 여부:
- 병합 확인 방법:
- 로컬 브랜치 최신화 여부:
- 작업 브랜치 정리 여부:
