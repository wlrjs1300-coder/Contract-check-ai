This document is a design draft, not an approval record. It records the proposed final design baseline for ContractCheck AI v0.2.2 based on the [integration review](v0.2.2-integration-review.md) of PR-1 through PR-5. It does not approve implementation, authentication, authorization, session, role-model operation, provider selection, endpoint selection, model selection, API key issuance, external AI use, real external transfer, security test execution, or production use. It is not legal advice, privacy compliance certification, security certification, or an operational approval record. Merging this document approves a design baseline only, not any of the above.

## 1. 문서 목적

이 문서는 [통합 검토 문서](v0.2.2-integration-review.md)에서 확인한 PR-1~PR-5의 정합성 검증 결과를 바탕으로 ContractCheck AI v0.2.2의 최종 설계 결정을 기록한다.

목적은 어떤 원칙이 설계 기준선(baseline)으로 채택 가능한지, 어떤 항목이 후속 구현 설계의 후보로 유지되는지, 어떤 항목이 추가 검토가 필요한 보류 상태인지, 어떤 항목이 현재 금지·미승인 상태인지를 네 가지 범주로 명확히 구분하는 것이다.

## 2. v0.2.2 목적

v0.2.2는 v0.2.1 핵심 기술 스파이크(조항 분할, 개인정보 탐지·마스킹, 마스킹 유용성, 출력 안전성, AI 공급자 정책 평가)의 결과를 실제 코드 구현 전에 하나의 개인정보·보안 통합 설계로 연결하는 문서 작업 단계이다. v0.2.2는 법률 자문, 개인정보보호 인증, 운영 승인 문서가 아니며 특정 AI 공급자를 선정하거나 외부 AI 사용을 승인하지 않는다.

## 3. 최종 설계 기준선

다음 원칙은 v0.2.2 설계 기준선으로 채택 가능하다고 판단한다. **이 채택은 설계 문서 수준의 baseline 확정이며, 실제 구현 승인이나 프로덕션 승인을 의미하지 않는다.**

## 6. DEFERRED / REVIEW REQUIRED

다음 항목은 추가 검토가 필요하며 v0.2.2 범위에서 확정하지 않는다.

- 인증 제품, MFA 범위, SSO 지원 여부
- session timeout, idle timeout 구체 값
- 구체 retention 기간(임시 보관 시간, D2/D3 보존 기간, 로그·감사 로그 보존 기간, 백업 보존 기간)
- confidence threshold, percentage 표시 여부
- expert review threshold 및 트리거-상태 매핑 규칙
- 사용자 고지 최종 문구 및 UI 배치
- 테스트 프레임워크, CI 실행 정책, secret scanning 통합 방식
- provider, endpoint, model 최종 선정
- 비용 한도, timeout 값, retry 횟수·간격·backoff 정책
- `source_hash` salt/pepper 및 해시 방식
- 로그·감사 무결성 구현 방식(hash chain, WORM, 서명)
- 삭제 재시도 횟수·간격, escalation 담당자
- D4(Public) 공개 정책 및 공개 전 검토 절차
- [통합 검토 문서](v0.2.2-integration-review.md) §5의 M-1~M-3 표기 불일치 정리

## 8. 범주 간 정합성 확인

§4~§7의 네 범주는 상호 배타적으로 작성되었다. 동일 항목이 두 범주에 동시에 속하는 사례는 없다. 예를 들어 "D1 외부 전송 금지"는 §4(기준선 원칙)에만 있으며, "provider 선정"은 §7(금지)에만 있고 §5(후보)에는 provider 관련 항목이 승인 게이트 형태로만 존재한다(§6, §9 참고).
