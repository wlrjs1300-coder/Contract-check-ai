# v0.7.2 위협 모델

## 문서 목적

계약서 업로드부터 분석, 저장, 삭제까지의 전체 데이터 경계에서 발생할 수 있는 위협을 식별하고, 현재 구현된 통제와 후속 구현이 필요한 통제를 분리한다.

현재 개발·검증 단계에서는 실제 개인정보 계약서 사용을 금지한다. 이 제한은 최종 제품의 영구 제한이 아니며, 보안 게이트와 별도 승인 절차를 모두 통과한 뒤 제한적 파일럿 허용을 검토한다.

## 평가 원칙

- 현재 구현된 통제만 `현재 통제`로 기록한다.
- 인증, ownership, 삭제 API, 저장 암호화, 관리자 감사, 실제 외부 Provider는 아직 구현되지 않았다.
- 불투명 식별자는 인증·인가 또는 IDOR 방어를 대체하지 않는다.
- Provider 실패 시 부분 결과를 저장하거나 노출하지 않고 atomic rollback 원칙을 유지한다.
- 실제 계약서, 실제 개인정보, 실제 Secret, 실제 외부 API 호출은 이번 문서 검증 범위에 포함하지 않는다.

## 1. IDOR 및 소유권 검증 누락

- 자산: 문서, 조항, 분석 작업, 분석 결과, 삭제 요청
- 공격자 또는 실패 주체: 다른 사용자, 권한 검사가 누락된 API 구현
- 공격·실패 경로: `document_id`, `job_id`, `result_id` 추측 또는 직접 입력, 누락된 ownership 검사
- 예상 영향: 다른 사용자 문서 조회·수정·삭제, 분석 결과 노출
- 현재 통제: 일부 불투명 식별자를 사용하지만 인증과 ownership 검증은 없으며 실질적인 IDOR 방어는 미구현
- 추가 통제: 인증, 모든 문서·작업·결과·삭제 API의 서버 측 ownership 검사, 관리자 권한과 감사, 존재 여부를 과도하게 노출하지 않는 오류 응답
- 잔여 위험: 신규 endpoint, batch·download·delete 경로, 관리자 우회 경로의 검사 누락
- 후속 버전: v0.7.4, 독립 검증 v0.7.8
- 검증 방법: 사용자 A가 사용자 B의 문서·작업·결과를 조회·수정·삭제할 수 없는지 순차 ID와 불투명 ID 모두로 테스트

## 2. 원본·임시 파일 잔존

- 자산: 업로드 원본, PDF 페이지 이미지, OCR 중간 이미지, 추출 파일, 임시 디렉터리
- 공격자 또는 실패 주체: cleanup 실패, 서버 재시작, 프로세스 비정상 종료, 운영자 실수
- 공격·실패 경로: 정상·실패·취소·timeout 경로의 cleanup 누락, symlink·junction·reparse point 우회
- 예상 영향: 계약 원문과 개인정보 잔존, 재식별, 로컬 경로 노출
- 현재 통제: request 단위 cleanup이 동작하며, v0.7.3에서 startup orphan sweeper가 추가됨
- 추가 통제: 모든 종료 경로 cleanup, 제한된 재시도, `original_cleanup_failed`, startup sweeper 또는 durable cleanup queue, 경로 우회 방어, 실제 경로 로그 금지
- 잔여 위험: TOCTOU, OS별 파일 잠금, 삭제 실패 후 장기 잔존
- 후속 버전: v0.7.4, 독립 검증 v0.7.8
- 검증 방법: 정상·실패·취소·timeout·재시작·비정상 종료 후 임시 파일과 디렉터리 미존재 확인

## 3. 추출 원문과 조항 원문의 평문 저장

- 예상 영향: 개인정보와 민감한 계약 조건 노출
- 현재 통제: 조항 원문 일부가 SQLite에 저장되며 저장 암호화와 사용자별 접근통제는 없음. 실제 개인정보 사용 금지로 위험을 제한
- 추가 통제: 일회성 모드 원문 영구 저장 제거, 인증·ownership·암호화 이후 보관형 모드 허용, 최소 보관기간, 사용자 삭제, backup 포함 데이터 정책
- 잔여 위험: migration·backup·debug dump에 원문 잔존
- 후속 버전: v0.7.3, v0.7.4, v0.7.5
- 검증 방법: 분석 완료 후 원문 파일·레코드 미존재 확인, DB dump와 backup의 원문 잔존 검사

## 4. 로그·예외·APM 유출

- 자산: 애플리케이션 로그, 오류 응답, stack trace, APM 이벤트
- 공격자 또는 실패 주체: 운영자, 외부 모니터링 서비스, 잘못된 로깅 설정
- 공격·실패 경로: request body 캡처, 원문·파일명·경로·Provider raw payload 출력

- 현재 통제: Provider raw payload·원문·경로를 로그에 남기지 않는 정책과 일부 테스트가 있으나 실제 운영 APM과 중앙 로그는 없음
- 추가 통제: 허용 메타 필드 allowlist, request body 수집 비활성, stack trace 외부 응답 차단, retention, 외부 APM 별도 승인
- 잔여 위험: 신규 logger·middleware·APM SDK 도입 시 설정 누락
- 후속 버전: v0.7.4, v0.9, 독립 검증 v0.7.8
- 검증 방법: 성공·실패·예외·timeout 시 로그와 오류 응답에서 본문·PII·Secret·경로 미노출 검사

## 5. PII false negative

- 자산: 이름, 주민등록번호, 외국인등록번호, 전화번호, 이메일, 주소, 계좌번호, 사업자등록번호 등
- 공격자 또는 실패 주체: 규칙 누락, OCR 오류, 문맥 변형, 탐지기 성능 한계
- 공격·실패 경로: 탐지되지 않은 개인정보가 마스킹되지 않고 외부 전송 후보가 됨
- 예상 영향: 외부 개인정보 전송, 규제·계약 위반, 사용자 신뢰 훼손
- 현재 통제: 규칙 기반 PII 탐지, 마스킹 후 잔여 PII 검사, 의심 시 fail-closed. 완전 탐지는 보장하지 않음
- 추가 통제: 독립 평가셋, 유형별 precision·recall·false negative 측정, 문맥 NER 또는 다중 탐지기 비교, 고위험 유형 별도 임계값, 낮은 신뢰도 외부 전송 차단
- 잔여 위험: 희귀 표현, 오탈자, OCR 왜곡, 표·이미지 속 개인정보
- 후속 버전: v0.7.8, 필요한 보완은 v0.7.6 이전
- 검증 방법: 유형별 합성 평가셋과 OCR 오류·띄어쓰기·오탈자·표 형식을 포함한 독립 평가

- 잔여 위험: 우회 표현, 부분 식별자, 문맥 기반 재식별
- 후속 버전: v0.7.6, v0.7.8
- 검증 방법: PII·token 변형·새 식별자를 포함한 Mock 응답의 전체 저장 차단 테스트

## 9. prompt injection

- 자산: 시스템 지침, 분석 계약, 출력 schema, Provider 실행 경계
- 공격자 또는 실패 주체: 악의적 문서 작성자, 계약서 안의 지시문
- 공격·실패 경로: 문서 데이터가 instruction으로 해석되거나 tool·URL·shell 실행을 유도
- 예상 영향: 분석 결과 조작, 정책 우회, 외부 동작 시도
- 현재 통제: 문서 텍스트를 데이터로 취급하는 Provider contract와 출력 검증이 있으나 실제 Provider prompt adapter는 없음
- 추가 통제: 시스템 지침과 계약서 데이터의 구조적 분리, 외부 tool·URL·shell 실행 금지, strict schema allowlist, 위험 키워드 차단만을 주 통제로 사용하지 않음
- 잔여 위험: 모델이 문서 지시를 따르는 비정형 우회
- 후속 버전: v0.7.6, v0.7.8
- 검증 방법: 이전 지침 무시, JSON 탈출, tool 호출, URL 접근, shell 실행 요구를 포함한 합성 문서 테스트

## 10. 악성 PDF·이미지·리소스 고갈·경로 조작

- 자산: 업로드 API, parser, OCR worker, 임시 저장소, CPU·메모리
- 공격자 또는 실패 주체: 외부 공격자, 손상 파일, parser 취약점
- 공격·실패 경로: PDF decompression bomb, 이미지 픽셀 폭탄, 과도한 OCR 작업, path traversal, symlink·junction·reparse point
- 예상 영향: 서비스 거부, 파일 덮어쓰기, 임의 경로 접근, 비용 증가
- 현재 통제: 확장자·MIME·시그니처·크기·페이지 제한, 저장소 밖 무작위 임시 경로, 일부 경로 검증. 압축 파일 업로드는 지원하지 않음
- 추가 통제: 이미지 픽셀 제한, OCR 시간·메모리 제한, parser sandbox 또는 별도 worker, 경로 우회 차단, 사용자별 rate limit
- 잔여 위험: parser zero-day, OS별 경로 동작 차이
- 후속 버전: v0.7.3, 독립 공격 검증 v0.7.8
- 검증 방법: 안전한 합성 fixture와 mock으로 크기·페이지·픽셀·시간·경로 제한 테스트

- 검증 방법: 삭제 후 활성 API·DB·파일·cache·mapping 검사와 backup 복원 후 재등장 방지 테스트

## 13. API Key·Secret 노출

- 자산: Provider key, OCR credential, DB credential, 암호화 키
- 공격자 또는 실패 주체: 개발자·운영자 실수, 로그·Git·APM 노출
- 공격·실패 경로: `.env` 커밋, 디버그 출력, 과도한 권한, 환경 혼용
- 예상 영향: 외부 서비스 악용, 비용 피해, 데이터 접근
- 현재 통제: 실제 Provider key 미사용, `.env`와 credential 파일 Git 제외, 환경변수 원칙, 공개 저장소 Secret 검사
- 추가 통제: 배포 Secret manager, 최소 권한, key rotation, 환경별 분리, 노출 시 즉시 폐기·재발급, 로그·APM 필터링
- 잔여 위험: 인적 실수, 서드파티 CI·배포 설정
- 후속 버전: v0.7.5, v0.9
- 검증 방법: Secret 패턴 스캔, 권한 검사, key rotation과 폐기 절차 점검

## 14. retry·비용 폭주

- 자산: Provider 비용, 분석 처리량, 사용자 응답시간
- 공격자 또는 실패 주체: 악의적 재요청, 장애 루프, 잘못된 retry 설정
- 공격·실패 경로: 과도한 재시도, 동시성 폭주, idempotency 누락
- 예상 영향: 비용 급증, 지연, 중복 결과
- 현재 통제: ProviderExecutionPolicy와 제한된 재시도 기반은 있으나 실제 Provider별 비용 정책은 없음
- 추가 통제: 사용자·문서·조직별 한도, timeout, max attempts, exponential backoff·jitter, circuit breaker, 비용 상한, 동시성 제한, idempotency
- 잔여 위험: 대규모 장애·트래픽 급증
- 후속 버전: v0.7.6, v0.9
- 검증 방법: 재시도·동시성·비용 상한·idempotency 시나리오와 장애 주입 테스트

---

## 15. Provider 장애와 부분 결과

- 자산: 분석 결과의 일관성, 기존 completed 결과, 작업 상태
- 공격자 또는 실패 주체: Provider 장애, timeout, malformed response
- 공격·실패 경로: 일부 finding만 반환, retry 중 중복 저장, 기존 결과 덮어쓰기
- 예상 영향: 불완전한 분석 노출, 사용자 오판, 결과 손상
- 현재 통제: 실패 시 atomic rollback, 부분 결과 저장 차단, 안전한 실패 상태와 오류 코드. 실제 외부 Provider는 없음
- 추가 통제: 실제 adapter 오류 매핑, idempotency, retry 후에도 부분 결과 저장 금지, 기존 completed 결과 덮어쓰기 금지, 장애 복구 절차
- 잔여 위험: 동기 처리와 서버 재시작 한계
- 후속 버전: v0.7.6, v0.9
- 검증 방법: timeout·rate limit·temporary failure·malformed response·부분 findings 주입 후 결과 0건과 기존 결과 보존 확인

---

## 16. 삭제 요청 누락

- 자산: 사용자의 삭제 권리, 문서·결과·backup 삭제 대상
- 공격자 또는 실패 주체: 삭제 API 오류, queue 유실, ownership 검사 누락
- 공격·실패 경로: 요청 접수 후 처리되지 않거나 다른 사용자 데이터가 삭제됨
- 예상 영향: 보관기간 초과, 사용자 신뢰 훼손, 오삭제
- 현재 통제: 사용자 삭제 API와 deletion 상태 모델은 구현되지 않았으며 v0.7.2에서 목표 상태만 정의
- 추가 통제: 인증된 삭제 API, ownership 검사, deletion 상태, 즉시 접근 차단, durable 삭제 queue, 삭제 확인 기록, 실패 알림·재시도, backup tombstone
- 잔여 위험: 장기 queue 지연, backup 반영 누락
- 후속 버전: v0.7.4, v0.7.5, v0.9
- 검증 방법: 정상·중복·실패·재시도·타 사용자 삭제 요청과 backup 복원 시나리오 E2E 테스트

## 17. 사용자 간 데이터 혼합

- 자산: 사용자별 문서, 임시 디렉터리, cache, Provider candidate
- 공격자 또는 실패 주체: 병렬 처리 오류, 잘못된 식별자 바인딩
- 공격·실패 경로: 임시 경로 공유, 사용자 scope 없는 DB query, cache key 충돌, candidate 참조 혼합
- 예상 영향: 다른 사용자 데이터가 결과나 evidence에 결합됨
- 현재 통제: 작업·문서 식별자 기반 분리는 있으나 인증·tenant·ownership 경계는 없음
- 추가 통제: 사용자별 ownership, 작업별 임시 디렉터리, DB query 사용자 scope, cache namespace, Provider request 문서·candidate 바인딩, 병렬 처리 격리
- 잔여 위험: 신규 비동기 worker·batch 경로의 바인딩 누락
- 후속 버전: v0.7.4, 독립 검증 v0.7.8
- 검증 방법: 병렬 업로드·분석에서 문서·조항·evidence·cache·임시 경로가 섞이지 않는지 테스트

## 19. 디버깅 중 원문 노출

- 자산: 원본 조항, 마스킹 전 텍스트, Provider raw payload, 실제 경로
- 공격자 또는 실패 주체: 개발자·운영자 실수, debug mode, 화면 캡처
- 공격·실패 경로: `print`, logger, 예외 message, debugger watch, 터미널·스크린샷
- 예상 영향: 계약 원문·개인정보·Secret 노출
- 현재 통제: 원문·마스킹 전 텍스트·raw Provider payload 로그 금지 규칙, 일부 보안 테스트와 공개 저장소 검사. 운영 APM은 없음
- 추가 통제: production debug 비활성, request body 캡처 금지, 민감 객체의 안전한 `repr`, 예외 message allowlist, 개발 환경 합성 데이터 강제, 화면 캡처·터미널 검토
- 잔여 위험: 로컬 수동 출력과 비공개 도구의 자동 수집
- 후속 버전: v0.7.4, v0.9, 독립 검증 v0.7.8
- 검증 방법: 디버그·예외·터미널·로그·스크린샷에 원문·PII·Secret·경로가 없는지 점검

## 후속 검증 기준

- 19개 위협 항목이 모두 존재한다.
- 각 항목에 자산, 공격자 또는 실패 주체, 공격·실패 경로, 예상 영향, 현재 통제, 추가 통제, 잔여 위험, 후속 버전, 검증 방법이 모두 존재한다.
- 인증·ownership·삭제 API·암호화·관리자 감사는 현재 구현으로 과장하지 않는다.
- 실제 Provider 위험은 v0.7.6, 키 관리는 v0.7.5, IDOR는 v0.7.4와 연결한다.
- atomic rollback과 부분 결과 비저장 원칙을 유지한다.
- 실제 개인정보 계약서 사용은 파일럿 보안 게이트 통과 전까지 금지한다.
