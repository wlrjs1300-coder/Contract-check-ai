## 1. 문서 목적

이 문서는 ContractCheck AI v0.2.2 개인정보·보안 통합 설계의 PR-1 계획 문서이다.

v0.2.1에서 확인한 조항 분할, 개인정보 탐지·마스킹, 마스킹 유용성, 출력 안전성, AI 공급자 정책 평가 결과를 하나의 개인정보·보안 설계 작업으로 연결한다. 실제 코드 구현 전에 데이터 흐름, 저장 범위, 외부 전송, 로그, 접근 통제, 실패 처리, 사용자 고지의 설계 범위와 산출물을 고정한다.

이 문서는 법률 자문, 개인정보보호 인증, 운영 승인 문서가 아니다. 또한 특정 AI 공급자를 선정하거나 외부 AI 사용을 승인하는 문서가 아니다.

PR-1에서는 v0.2.2 전체 범위, 제외 범위, 필수 산출물, PR 분할, 승인 게이트를 고정했다. PR-2에서는 데이터 분류, 개인정보 처리 흐름, 신뢰 경계 초안을 작성해 GitHub에 반영했다. 현재 PR-3에서는 저장·보존·삭제·로그·감사 초안을 작성 중이며, 이 초안은 독립 검토와 사용자 승인 전까지 최종 기준이 아니다.

## 2. 배경과 v0.2.1 결과

v0.2.1 핵심 기술 스파이크는 실제 제품 구현이 아니라 후속 설계의 불확실성을 줄이기 위한 검증 단계였다. PR-1부터 PR-6까지 모두 완료됐으며, v0.2.2는 그 결과를 개인정보·보안 설계 기준으로 통합한다.

| PR | 주제 | v0.2.2에 연결할 결과 | 안전 경계 |
|---|---|---|---|
| PR-2 | 조항 분할 | synthetic TXT 기준 조항 분할 검증 완료, `reference_id`, `source_hash`, section overlap, uncovered non-whitespace 검증 확인 | 실제 계약서 처리 승인 아님 |
| PR-3 | 개인정보 탐지·마스킹 | fixture 4/4, entity 26/26, exclusion 15/15, residual 0, PASS | 실제 개인정보 처리 승인 아님 |
| PR-4 | 마스킹 유용성 | 마스킹 후 분석 유용성 PASS, corrective 검증 포함, residual 0 | 실제 외부 전송 승인 아님 |
| PR-5 | 출력 안전성 | `ALLOW`, `BLOCK`, `REVIEW` 판정 흐름 검증, 법률 확정·보장 표현 차단 방향 확인 | 제품 운영 승인 아님 |
| PR-6 | AI 공급자 정책 평가 | OpenAI API `CONDITIONAL`, Anthropic API `CONDITIONAL`, Google Gemini API `REVIEW`, 최종 판단 B 기록 | 공급자 선정 없음, 외부 AI 사용 승인 없음 |

PR-6의 최종 판단은 다음과 같다.

```text
B. External AI use requires additional security and policy review before product use.
```

Gemini API Unpaid Services는 계약서 분석에 사용하지 않는다. OpenAI API, Anthropic API, Google Gemini API paid path 중 어느 것도 현재 공급자로 선정되지 않았다.

## 6. 데이터 분류 후보

PR-2에서는 다음 데이터 후보를 분류 대상으로 삼는다. 각 데이터의 실제 등급, 저장 가능 여부, 외부 전송 가능 여부는 PR-2 초안에서 제안하고 독립 검토와 사용자 승인 전까지 확정하지 않는다.

- original contract file
- extracted text
- clause data
- source hash
- reference_id
- detected personal data
- masked clause data
- residual scan result
- outbound candidate payload
- provider response
- normalized analysis result
- display label
- confidence score
- expert review recommendation
- job status
- audit metadata
- operational logs
- deletion metadata
- policy/model/endpoint version

`confidence score`는 기술적 분석 산출물 후보로만 기록한다. 사용자에게 위험도 숫자, 점수, 퍼센트로 노출할지 여부는 이번 문서에서 확정하지 않는다.
