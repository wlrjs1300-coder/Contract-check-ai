# v0.7.1 Provider PII Gate

## 기본 정보

- 버전: v0.7.1
- 브랜치: `feat/v0.7.1-provider-pii-gate`
- 시작일: 2026-07-21
- 완료일: 2026-07-22
- 현재 단계: 완료

## 목표

확정된 조항 텍스트에 대해 개인정보 탐지·마스킹을 수행한 뒤에도
Provider 요청에 PII가 남지 않도록 차단하고, Provider 요청/응답 경계를 최소화한다.
또한 분석 출력에서 PII 재생성 의심 문자열을 검출해 저장을 막는다.

## 보안 경계

- 조항 본문은 `detect_and_mask`로 마스킹 후 Provider에 전달한다.
- 마스킹 결과에 잔여 PII가 있으면 즉시 `provider_data_minimization_failed` 실패.
- 마스킹 토큰 재사용 충돌이 감지되면 즉시 `provider_redaction_token_collision` 실패.
- Provider 요청은 `masked` 텍스트와 evidence 후보 최소 정보만 포함.
- `document_id`는 내부 원본 ID 대신 불투명 `doc-<sha256 prefix>`로 전송.
- 요청 payload는 경로 문자열 패턴(`\`/`:`/`..`) 유입 시 차단.
- Provider 출력은 `check_summary_output` + PII 잔여 필드 스캔으로 점검 후 반영.
- Provider 출력에서 PII가 감지되면 `provider_output_pii_detected` 실패.

## 에러 코드

- `provider_redaction_token_collision`
- `provider_data_minimization_failed`
- `provider_output_pii_detected`
- `provider_output_pii_reconstruction_suspected` (운영에서 상세 세부 이유로 준비)
- `provider_output_redaction_token_invalid` (운영에서 정합성 검사 시 활용)

## 금지 항목

- 원본 clause 본문/파일 경로/파일명/원본 bytes 원형 저장
- raw Provider request/response 재저장 및 raw 값 노출 로그
- API 응답/예외 메시지에서 token 값·상세 PII 값 노출

## v0.7.1 범위

- 실제 외부 Provider SDK 연동은 v0.7.6 비식별 데이터 기반 실제 Provider로 이월.
- PII 탐지 정밀도 완전 보장은 별도 단계.
- token 정책 변경(포맷, prefix 확장) 승인 없는 임의 조정 금지.
