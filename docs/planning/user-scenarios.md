# ContractCheck AI 사용자 시나리오

## 1. 문서 목적과 범위

이 문서는 v0.1.4 사용자 시나리오 산출물이다.

v0.1.1 서비스 개요, v0.1.2 MVP 범위, v0.1.3 지원 계약서 범위를 바탕으로 사용자가 ContractCheck AI를 이용하는 정상 흐름, 예외 흐름, 실패 흐름, 재시도 방향을 정리한다.

이번 문서는 사용자의 행동과 서비스의 안내 방향을 정의한다. 화면명, 버튼명, UI 레이아웃, API 경로, HTTP 상태 코드, DB 테이블과 필드, JWT 저장 방식, 파일 크기 제한 수치, timeout과 retry 횟수, 개인정보 탐지 상세 규칙, 마스킹 형식, AI 프롬프트, AI 응답 JSON 스키마, 계약서 유형 판별 알고리즘, 최종 고지와 면책 문구는 확정하지 않는다.

분석 결과는 법률 자문이나 최종 판단이 아니다. 서비스는 적법성, 위법성, 무효, 법적 효력, 승소 가능성, 금전 반환, 해고 결과를 확정하지 않는다.

## 2. 사용자 역할

| 역할 | 가능한 행동 | 불가능한 행동 | 확인할 보안 경계 |
|---|---|---|---|
| 비회원 | 회원가입, 로그인 진입 | 계약서 업로드, 분석 요청, 분석 결과 조회 | 인증 전 계약서와 분석 결과에 접근하지 못한다. |
| 인증된 일반 사용자 | 계약서 업로드, 분석 요청, 본인 분석 상태 확인, 본인 결과 조회, 본인 분석 이력 재조회, 로그아웃 | 다른 사용자 결과 조회, 관리자 기능 사용 | 본인 데이터만 접근할 수 있어야 한다. |
| 시스템 처리 주체 | 파일 지원 여부 확인, 텍스트 추출, 개인정보 탐지와 마스킹, 조항 분리, 분석 상태 갱신, 원본 삭제 처리 | 법률 판단 확정, 사용자 대신 계약서 수정 | 원문과 마스킹 전 텍스트를 로그에 남기지 않는다. |
| 관리자 | 없음 | 사용자 관리, 전체 분석 조회, 수동 검토, 운영자 대시보드 | 관리자 기능은 MVP에서 제외한다. |

| N-07 | 텍스트 추출 | 지원 파일임 | 분석을 요청한다. | PDF에서 텍스트를 추출한다. | 분석 입력 텍스트를 준비한다. | 개인정보 처리 |
| N-08 | 개인정보 탐지와 마스킹 | 텍스트가 추출됨 | 처리 진행을 기다린다. | 개인정보 포함 가능 영역을 탐지하고 마스킹한다. | 마스킹된 텍스트가 분석 준비에 사용된다. | 조항 분리 |
| N-09 | 분석 진행 | 조항 분리 가능 | 분석 상태를 확인한다. | 분석 대기, 진행 중, 완료, 실패 상태를 갱신한다. | 사용자가 현재 상태를 알 수 있다. | 결과 확인 |
| N-10 | 결과 확인 | 분석 완료 | 결과를 확인한다. | 관련 문구, 핵심 확인, 추가 확인, 조건 확인, 전문가 검토 권고 여부를 보여준다. | 사용자가 상담 전 확인할 항목을 정리한다. | 이력 조회 또는 로그아웃 |
| N-11 | 분석 이력 재조회 | 완료된 본인 분석 결과가 있음 | 본인 분석 목록에서 결과를 다시 연다. | 본인 결과만 보여준다. | 사용자가 완료 결과를 다시 확인한다. | 로그아웃 |
| N-12 | 로그아웃 | 사용자가 로그인 상태임 | 로그아웃한다. | 인증 상태를 종료한다. | 다른 사람이 같은 환경에서 결과에 접근하기 어려워진다. | 종료 |

## 4. 인증·접근 권한 시나리오

| ID | 상황 | 사용자 안내 방향 | 접근 허용 여부 | 후속 설계 |
|---|---|---|---|---|
| A-01 | 비회원이 업로드를 시도 | 로그인이 필요하다고 안내한다. | 불가 | v0.3, v0.4 |
| A-02 | 비회원이 분석 결과 URL에 접근 | 인증 후 본인 결과만 확인할 수 있다고 안내한다. | 불가 | v0.3, v0.4 |
| A-03 | 인증 사용자가 본인 결과에 접근 | 본인 분석 결과를 보여준다. | 가능 | v0.4 |
| A-04 | 인증 사용자가 다른 사용자 결과에 접근 | 접근할 수 없다고 안내한다. | 불가 | v0.4 |
| A-05 | 로그인 상태가 만료됨 | 다시 로그인하도록 안내한다. | 불가 | v0.2, v0.4 |
| A-06 | 회원가입 실패 | 가입이 완료되지 않았음을 안내하고 다시 시도할 수 있게 한다. | 불가 | v0.3, v0.4 |
| A-07 | 로그인 실패 | 로그인에 실패했음을 안내하고 다시 시도할 수 있게 한다. | 불가 | v0.3, v0.4 |
| A-08 | 인증 상태 상실 | 인증이 유지되지 않아 다시 로그인해야 함을 안내한다. | 불가 | v0.2, v0.4 |

비회원은 업로드, 분석 요청, 결과 조회를 할 수 없다. 인증된 일반 사용자만 계약서를 업로드하고 본인 결과를 조회할 수 있다.

회원가입 실패, 로그인 실패, 인증 만료 또는 인증 상태 상실 상황에서는 실패 사실과 재시도 또는 재인증 방향만 안내한다. 이메일 중복 판정 규칙, 비밀번호 정책, 오류 코드, 토큰 정책은 이번 단계에서 확정하지 않는다.

업로드 네트워크 중단은 사용자 취소와 구분되는 비자발적 중단이다. 업로드 완료 전이면 분석을 시작하지 않는 방향으로 둔다. 임시 파일 정리와 재시도 정책은 v0.2 또는 v0.5에서 확정한다.

| S-07 | 개인정보 처리 실패 | 개인정보 보호 처리가 완료되지 않아 분석할 수 없다고 안내한다. | 가능 | 실패 상태 표시 |
| S-08 | 분석 응답 형식 오류 | 결과를 표시할 수 없다고 안내한다. | 가능 | 실패 상태 표시 |
| S-09 | 분석이 장시간 끝나지 않음 | 지연 가능성을 안내한다. | 후속 설계 | 상태 정책 미확정 |
| S-10 | 사용자가 페이지를 벗어남 | 이력에서 상태를 다시 확인할 수 있게 한다. | 해당 없음 | 상태 유지 방향 |
| S-11 | 사용자가 나중에 다시 접속 | 본인 분석 이력에서 상태를 확인한다. | 가능 | 표시 가능 |
| S-12 | 사용자가 분석 진행 중 취소 | 사용자가 취소한 결과로 안내한다. | 가능 | 실패와 구분되는 취소 결과 표시 방향 |

timeout 수치, retry 횟수, 비동기 작업 큐 도입 여부, 작업 복구 정책은 이번 단계에서 확정하지 않는다.

분석 진행 중 사용자 취소는 일반적인 분석 실패와 구분되는 사용자 결과로 정의한다. 사용자에게 임의로 분석 실패라고 표시하지 않는다. 최종 저장 상태를 독립적인 취소 결과로 둘지, 별도 처리 결과나 실패 사유로 저장할지는 v0.4 또는 v0.5에서 확정한다.

## 9. 분석 결과 확인

| ID | 상황 | 사용자에게 보여줄 정보 | 표현 경계 |
|---|---|---|---|
| R-01 | 정상 결과 확인 | 관련 문구, 핵심 확인, 추가 확인, 조건 확인, 권장 행동, 전문가 검토 권고 여부 | 법률 자문이나 최종 판단으로 표현하지 않는다. |
| R-02 | 일부 조항 결과 누락 | 일부 결과를 표시하지 못했을 수 있음을 안내한다. | 누락이 안전함을 의미하지 않는다. |
| R-03 | 관련 문구 연결 실패 | 결과와 원문 연결을 확인할 수 없다고 안내한다. | 임의 연결을 하지 않는다. |
| R-04 | 전문가 검토 권고가 있음 | 전문가 검토가 필요할 수 있음을 안내한다. | 전문가 연결 기능을 제공한다고 표현하지 않는다. |
| R-05 | 전문가 검토 권고가 없음 | 별도 권고가 없음을 표시할 수 있다. | 안전하거나 문제가 없다고 보장하지 않는다. |
| R-06 | 사용자가 확정 답변을 기대함 | 참고용 결과임을 안내한다. | 적법성, 위법성, 무효, 법적 효력, 승소, 반환, 해고 결과를 확정하지 않는다. |

결과에는 위험도 수치나 퍼센트를 사용하지 않는다. 행동 중심 표현은 `핵심 확인`, `추가 확인`, `조건 확인`, `전문가 검토 권고`를 우선 사용한다.

`핵심 확인`, `추가 확인`, `조건 확인`, `전문가 검토 권고`는 `docs/planning/mvp-scope.md`의 "위험도 또는 검토 우선순위"를 비수치적으로 표현하기 위한 후보이다. 최종 단계명과 표시 방식은 v0.3 화면 설계와 v0.5 분석 파이프라인 설계에서 확정한다.

## 10. 분석 이력과 재조회

| ID | 상황 | 사용자 안내 방향 | 접근 제어 |
|---|---|---|---|
| H-01 | 본인 분석 목록 조회 | 본인 분석 이력을 보여준다. | 본인만 접근 가능 |
| H-02 | 완료 결과 재조회 | 완료된 분석 결과를 다시 보여준다. | 본인만 접근 가능 |
| H-03 | 진행 중 작업 재조회 | 현재 상태를 보여준다. | 본인만 접근 가능 |
| H-04 | 실패 작업 재조회 | 실패 상태와 최소 정보를 보여준다. | 본인만 접근 가능 |

실패한 분석은 최소 정보와 실패 상태를 이력에 표시한다. 실패 상세 원인, 보관 기간, 삭제 가능 여부는 후속 설계에서 정한다.

## 15. v1.0 최초 구현 범위와 후속 확장 경계

전체 서비스 지원 계획은 근로계약서와 주택 임대차계약서를 유지한다.

다만 v1.0 최초 구현과 기술 검증 우선 대상은 근로계약서 1종으로 둔다. 근로계약서를 통해 업로드, 텍스트 추출, 개인정보 처리, 조항 분리, 분석 결과 표시, 이력 재조회 흐름을 먼저 검증한다.

주택 임대차계약서는 후속 구현 대상으로 둔다. v0.1 기획 문서에서 정의한 주택 임대차계약서 지원 범위는 유지하되, 실제 구현 순서는 기술 검증과 MVP 완성도를 고려해 단계적으로 진행한다.

v1.0 최초 구현 시 주택 임대차계약서 선택지를 숨김, 비활성화, 준비 중 안내 중 어떤 방식으로 표시할지는 v0.3 화면 설계에서 확정한다.

| 구분 | 범위 |
|---|---|
| v1.0 최초 구현 우선 | 근로계약서 1종 |
| 후속 구현 대상 | 주택 임대차계약서 |
| 유지되는 서비스 계획 | 근로계약서와 주택 임대차계약서 지원 |
