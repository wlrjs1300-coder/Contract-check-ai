# ContractCheck AI 서비스 책임·고지 정책

## 1. 문서 목적과 적용 범위

이 문서는 v0.1.5 서비스 책임·고지 정책 산출물이다.

ContractCheck AI가 사용자에게 어떤 수준의 도움을 제공하고, 어떤 판단을 제공하지 않으며, 개인정보와 원본 파일 처리에 대해 어떤 방향으로 안내해야 하는지 정책 수준에서 정리한다.

이번 문서는 최종 약관, 고지 문구, 동의 문구, 화면 문구를 확정하지 않는다. 구체적인 화면 위치, 버튼명, 체크박스 문구, API, DB, HTTP 코드, JWT 정책, timeout과 retry 수치, AI 프롬프트, AI 응답 JSON 스키마, 출력 금지 표현 필터 구현 방식은 후속 단계에서 다룬다.

## 2. 서비스의 역할

ContractCheck AI는 계약서를 업로드하기 전에 주요 확인 항목을 정리하는 계약서 사전점검 참고 도구이다.

서비스는 사용자가 계약서의 관련 문구, 모호한 표현, 추가 확인이 필요한 조건, 전문가에게 질문할 항목을 정리하도록 돕는다.

서비스는 법률 전문가를 대체하지 않는다. 변호사, 노무사, 공인중개사 등 전문가의 법률 검토나 상담을 대신하지 않으며, 사용자가 전문가 상담 전에 질문을 준비하는 데 도움을 주는 수준으로 한정한다.

## 3. 서비스가 제공하지 않는 것

ContractCheck AI는 다음을 제공하지 않는다.

- 법률 자문
- 계약서 또는 조항의 적법성 확정
- 계약서 또는 조항의 위법성 확정
- 계약서 또는 조항의 무효 여부 확정
- 계약 효력 보장
- 계약 체결 또는 해지 결정
- 소송에서의 승소 가능성 보장
- 보증금, 임금, 손해배상 등 금전 반환 보장
- 해고, 퇴직, 계약 종료 결과 보장
- 모든 위험과 누락 탐지 보장
- 전문가 검토 대체
- 계약서 자동 수정
- 전문가 자동 연결 보장

서비스는 사용자의 계약 판단을 대신하지 않는다. 최종 계약 체결, 서명, 해지, 분쟁 대응 여부는 사용자와 전문가의 검토 영역이다.

## 4. AI 분석 한계

AI 분석 결과에는 오류, 누락, 잘못된 연결, 문맥 오해가 포함될 수 있다.

같은 문서라도 분석 시점, 입력 품질, 조항 분리 결과, 모델 응답 상태에 따라 결과가 달라질 수 있다.

전문가 검토 권고가 표시되지 않은 조항도 안전하다는 뜻이 아니다. 별도 권고가 없다는 것은 해당 결과 안에서 권고가 표시되지 않았다는 의미로 제한한다.

분석 결과는 계약 체결, 계약 해지, 분쟁 대응, 소송 대응의 단독 판단 근거로 사용하지 않도록 안내해야 한다.

## 5. 분석 결과 표현 원칙

분석 결과에는 위험도 숫자, 점수, 퍼센트를 사용하지 않는다.

사용자에게는 다음과 같은 행동 중심 표현을 우선 사용한다.

- 핵심 확인
- 추가 확인
- 조건 확인
- 전문가 검토 권고

`핵심 확인`, `추가 확인`, `조건 확인`, `전문가 검토 권고`는 `docs/planning/mvp-scope.md`의 "위험도 또는 검토 우선순위"를 점수나 퍼센트가 아닌 행동 중심의 비수치 방식으로 표현하기 위한 후보 체계이다.

분석 결과는 가능성과 확인 필요성을 안내하는 표현으로 제한한다. 법률적으로 확정하거나 보장하는 것처럼 읽히는 표현은 사용하지 않는다.

사용하지 않는 표현의 예시는 다음과 같다.

- 적법
- 위법
- 무효
- 법적 효력 보장
- 법적으로 안전
- 승소 보장
- 반환 보장
- 해고 결과 확정
- 모든 위험 탐지
- 완전한 보호

최종 단계명, 분류 기준, 표시 순서, 화면 배치, 구체 문구는 v0.3 화면 설계와 v0.5 분석 파이프라인 설계에서 확정한다.

## 6. 출력 안전성 검증 방향

AI 분석 결과에는 금지 표현, 법률 확정 표현, 보장 의미, 과도한 단정 표현이 포함될 가능성이 있다.

서비스는 안전성 검증을 통과하지 못한 결과를 사용자에게 그대로 노출하지 않는 방향을 원칙으로 둔다.

출력 안전성 검증에서는 다음 가능성을 확인해야 한다.

- 법률 자문처럼 읽히는 표현
- 적법성, 위법성, 무효 여부를 확정하는 표현
- 계약 효력, 승소, 금전 반환, 해고 결과를 보장하는 표현
- 개인정보 또는 계약서 원문 노출 가능성
- 사용자에게 모든 위험을 탐지했다고 오해하게 만드는 표현

차단, 재작성, 재시도 또는 실패 처리의 구체적인 구현 방식은 기술 스파이크와 v0.5 분석 파이프라인 설계에서 확정한다.

## 7. 개인정보 처리 고지 방향

서비스는 AI 분석 전에 개인정보 탐지와 마스킹을 수행하는 방향을 기본 원칙으로 둔다.

개인정보 보호 원칙은 사용자 본인 정보뿐 아니라 업로드 문서에 포함된 모든 개인정보에 적용한다. 계약서에는 계약 상대방, 회사 담당자, 고용주, 임대인, 임차인 등 제3자의 개인정보가 포함될 수 있으므로, 제3자 개인정보도 개인정보 탐지와 마스킹 대상으로 고려한다.

개인정보 완전 탐지를 보장하지 않는다. 탐지되지 않은 개인정보가 남을 가능성, 잘못 탐지되는 가능성, 일부만 마스킹되는 가능성을 인정한다.

개인정보 탐지 또는 마스킹에 실패한 경우에는 분석을 진행하지 않는 방향을 우선한다.

마스킹 전 계약서 원문을 외부 AI 분석 단계에 전달하지 않는 것을 원칙으로 둔다.

AI 공급자에게 전송되는 데이터는 마스킹된 분석 입력으로 제한하는 방향을 유지한다.

마스킹된 분석 입력이 외부 AI 공급자에게 전달될 수 있음을 사용자에게 알려야 한다. 외부 AI 공급자의 정책에 따라 전송 데이터가 일정 기간 저장되거나 서비스 개선 등에 활용될 가능성도 고지 대상에 포함한다.

실제 저장 여부, 저장 기간, 학습 또는 서비스 개선 활용 여부, 보관 비활성화 가능 여부는 공급자별 정책 확인 후 확정한다. 아직 실제 공급자가 정해지지 않았으므로 특정 공급자가 데이터를 실제로 저장하거나 학습한다고 단정하지 않는다.

실제 서비스에서는 확정된 공급자 정책을 기준으로 사용자에게 정확한 내용을 알려야 한다. 정책을 확인하지 못했거나 ContractCheck AI의 개인정보 보호 기준에 맞지 않는 공급자는 사용하지 않는 방향을 유지한다.

사용자는 자신이 업로드할 수 있는 문서인지 확인하고, 분석에 불필요한 제3자 개인정보의 업로드를 최소화해야 한다. 다만 제3자 개인정보가 포함됐다는 이유로 개인정보 보호와 안전한 처리 책임을 사용자에게 전부 이전하지 않는다.

사용자가 제3자의 동의 또는 별도 법적 권한을 반드시 확보했다고 단정하지 않는다. 권한이 없거나 업로드하면 안 되는 문서를 업로드하지 않도록 안내하는 방향을 둔다.

구체적인 업로드 권한, 제3자 동의 요건, 이용약관, 개인정보 처리방침 문구, AI 공급자 정책 확인은 v0.2 개인정보·보안 설계 또는 기술 스파이크에서 확정한다.

## 8. 원본 파일 처리 고지

서비스는 업로드 원본 파일을 영구 저장하지 않는 방향을 기본 원칙으로 둔다.

원본 파일 처리 방향은 다음과 같다.

| 상황 | 처리 고지 방향 |
|---|---|
| 정상 분석 완료 | 처리 완료 후 원본 파일을 삭제하는 방향을 안내한다. |
| 분석 실패 | 실패 후에도 원본 파일 보호와 삭제 방향을 유지한다고 안내한다. |
| 사용자 취소 | 취소된 작업의 원본 파일 처리 방향을 안내한다. |
| 삭제 실패 | 삭제 실패 가능성과 후속 처리 필요성을 숨기지 않는다. |

완벽한 삭제를 보장하지 않는다. 삭제 실패 가능성을 숨기거나 사용자에게 모든 파일이 항상 즉시 삭제된다고 표현하지 않는다.

삭제 실패 시 사용자 안내와 후속 처리 방향이 필요하다.

삭제 시점, 삭제 재시도 횟수, 임시 보관 시간, 정리 작업 방식, 감사 로그 범위는 v0.2 개인정보·보안 설계 또는 v0.5 분석 파이프라인 설계에서 확정한다.

## 9. 고지가 필요한 시점

고지는 사용자가 서비스를 오해하지 않도록 필요한 순간에 제공한다.

정책 수준에서 고지가 필요한 시점은 다음과 같다.

- 계약서 업로드 전
- 분석 요청 전
- 분석 결과 확인 시
- 전문가 검토 권고 표시 주변
- 분석 실패 발생 시
- 개인정보 탐지 또는 마스킹 실패 발생 시
- 결과 안전성 검증 실패 발생 시
- 원본 파일 삭제 실패 발생 시
- 사용자 취소 또는 장시간 분석 지연 발생 시

사용자가 최소한 최초 분석 결과를 확인할 때는 결과의 참고용 성격과 주요 한계를 인지할 수 있어야 한다.

결과를 다시 조회할 때마다 같은 고지를 반복 노출할지, 동일 세션에서 한 번만 보여줄지는 v0.3 화면·UX 설계에서 결정한다.

결과의 핵심 한계를 사용자가 쉽게 무시할 수 있는 위치에만 배치하지 않는 방향을 유지한다. 동시에 반복적인 면책 문구로 결과 이용을 과도하게 방해하지 않도록 한다.

회원가입 약관 전체, 화면 배치, 실제 문구 위치, UI 형태는 v0.3 화면 설계에서 결정한다.

## 10. 사용자 동의 방향

사용자는 업로드 및 분석 전에 서비스 한계와 개인정보 처리 방향을 확인할 수 있어야 한다.

사용자는 ContractCheck AI가 법률 자문 서비스가 아니라 계약서 사전점검 참고 도구라는 점을 인지할 수 있어야 한다.

동의는 다음 내용을 이해할 수 있도록 구성하는 방향을 둔다.

- 분석 결과가 최종 법률 판단이 아니라는 점
- 개인정보 탐지와 마스킹이 수행되지만 완전 탐지를 보장하지 않는다는 점
- 마스킹된 분석 입력이 외부 AI 분석 단계에 전달될 수 있다는 점
- 외부 AI 공급자의 확인된 데이터 저장, 보관, 학습 또는 서비스 개선 활용 정책
- 업로드 문서에 사용자 본인뿐 아니라 제3자 개인정보가 포함될 수 있다는 점
- 권한이 없거나 업로드하면 안 되는 문서를 업로드하지 않아야 한다는 점
- 원본 파일을 영구 저장하지 않는 방향이지만 완벽한 삭제를 보장하지 않는다는 점
- 실패, 취소, 지연 상황에서 결과가 제공되지 않을 수 있다는 점

구체적인 체크박스, 버튼명, 약관 문구, 동의 기록 방식은 v0.3 화면 설계와 v0.4 DB/API 설계에서 확정한다.

미성년자 이용, 보호자 또는 대리인의 업로드, 본인 계약서가 아닌 문서의 업로드 가능 범위는 실제 서비스 약관과 개인정보 정책 설계에서 검토한다. 이번 단계에서는 이용 가능 연령, 법정대리인 동의 방식, 대리 권한 확인 절차를 확정하지 않는다.

## 11. 결과 사용 책임

사용자는 분석 결과를 참고 자료로 활용한다.

분석 결과는 계약서 이해, 추가 확인 질문 정리, 전문가 상담 준비를 돕기 위한 정보이다.

최종 계약 판단, 계약 체결, 계약 해지, 분쟁 대응, 소송 대응은 사용자와 전문가의 검토 영역이다.

서비스 책임을 과도하게 면제하거나 사용자에게 모든 책임을 전가하는 문구는 사용하지 않는다. 서비스의 한계를 명확히 알리되, 개인정보 보호와 안전한 결과 표시를 위한 서비스의 관리 책임도 함께 유지한다.

사용자는 업로드 문서에 포함된 정보와 업로드 가능 여부를 확인해야 하지만, 서비스는 개인정보 보호와 안전한 처리 책임을 사용자에게 전부 넘기지 않는다. 제3자 개인정보가 포함될 가능성을 고려해 탐지, 마스킹, 고지, 실패 처리 방향을 함께 유지한다.

## 12. 실패 상황 고지

실패 상황에서는 사용자가 무엇이 발생했는지 최소한 이해할 수 있어야 한다. 다만 이번 단계에서는 실제 화면 문구를 확정하지 않는다.

| 상황 | 최소 고지 방향 |
|---|---|
| 분석 실패 | 분석이 완료되지 않았고 다시 시도할 수 있는지 안내한다. |
| 개인정보 탐지 실패 | 개인정보 보호 처리가 완료되지 않아 분석을 진행하지 않는 방향임을 안내한다. |
| 마스킹 실패 | 마스킹 전 텍스트를 분석 단계에 전달하지 않는 방향임을 안내한다. |
| 결과 안전성 검증 실패 | 결과를 그대로 표시할 수 없어 실패 또는 재처리가 필요할 수 있음을 안내한다. |
| 원본 파일 삭제 실패 | 삭제 실패 가능성과 후속 처리 필요성을 안내한다. |
| 장시간 분석 지연 | 분석이 지연되고 있음을 안내한다. |
| 사용자 취소 | 사용자가 취소한 결과로 안내하고 일반적인 분석 실패와 구분한다. |

실패 원인 상세, 재시도 조건, 상태 저장 방식, 오류 응답 구조는 v0.4 또는 v0.5에서 확정한다.
