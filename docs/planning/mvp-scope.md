# ContractCheck AI MVP 범위

## 1. 문서 목적

이 문서는 v0.1.2 MVP 범위 산출물이다.

ContractCheck AI의 포트폴리오용 MVP에서 반드시 포함할 기능, 선택적으로 포함할 기능, 이번 MVP에서 제외할 기능을 구분하고 후속 설계 단계와의 경계를 명확히 한다.

본 문서는 서비스 기획 범위 문서이며, 화면 설계, API 명세, DB 설계, AI 프롬프트, 구체적인 보안 정책 수치는 후속 단계에서 다룬다.

## 2. MVP 목표

ContractCheck AI MVP는 사용자가 계약서를 업로드하기 전에 주요 조항을 이해하고 위험 가능성과 추가 확인이 필요한 항목을 정리할 수 있도록 돕는 참고용 AI 계약서 사전점검 서비스를 구현하는 것을 목표로 한다.

MVP는 법률 자문, 계약의 적법성 판단, 전문가 검토 대체가 아니라 사용자가 계약서를 더 잘 이해하고 상담 전 질문을 준비하도록 돕는 제품 경험에 집중한다.

## 3. MVP 핵심 사용자 흐름

1. 사용자가 이메일과 비밀번호로 가입하거나 로그인한다.
2. 사용자가 텍스트 레이어가 있는 PDF 계약서를 업로드한다.
3. 서비스가 업로드 파일이 MVP 지원 형식인지 확인하고, 지원하지 않는 경우 사용자에게 안내한다.
4. 서비스가 계약서에서 텍스트를 추출한다.
5. 서비스가 AI 분석 전에 개인정보를 탐지하고 마스킹한다.
6. 서비스가 마스킹 완료 후 계약서 내용을 조항 단위로 나눈다.
7. 서비스가 조항별 위험 가능성과 확인 필요 사항을 AI로 분석한다.
8. 사용자가 분석 진행 상태를 확인한다.
9. 사용자가 분석 결과에서 관련 조항, 위험 가능성 설명, 추가 질문, 권장 행동을 확인한다.
10. 사용자가 자신의 분석 이력에서 완료된 결과를 다시 조회한다.
11. 처리 완료 후 업로드 원본 파일은 영구 저장하지 않고 삭제하는 방향으로 운영한다.

## 6. MVP 선택 기능

| 항목 | 최소 범위 | 선택으로 둔 이유 |
|---|---|---|
| 기본 상태 필터 | 분석 상태의 구분은 필수 기능이며, 사용자가 진행 중, 완료, 실패 항목만 따로 필터링해 보는 기능은 선택 기능으로 둔다. | 상태별 필터 UI는 이력 사용성을 높이지만 핵심 분석 흐름의 필수 조건은 아니다. |
| 사용자 지정 문서 표시명 | 사용자가 개인정보가 포함되지 않은 문서 표시명을 입력할 수 있다. | 원본 파일명을 그대로 노출하지 않는 데 도움이 되지만 세부 정책은 개인정보 설계와 함께 확정해야 한다. |
| 간단한 인쇄 친화 보기 | 분석 결과를 읽기 쉬운 형태로 출력하거나 브라우저 인쇄에 적합하게 보여줄 수 있다. | 상담 준비에 유용하지만 파일 내보내기나 리포트 생성 기능까지 확장하지 않는다. |

## 7. MVP 제외 기능

### 서비스 확장

- 결제
- 관리자 페이지
- 기업 또는 조직 계정
- 다중 권한 체계
- 통계 대시보드
- 대량 계약서 처리
- ERP 수준의 계약 관리 기능

### 계약 관련

- 법률 자문
- 계약의 적법성 또는 무효 여부 최종 판단
- 계약 체결 여부 결정
- 계약서 자동 작성
- 전자서명
- 소송 결과 예측
- 전문가 역할 대체

### 문서 처리

- OCR
- 스캔 PDF 분석
- 이미지 기반 PDF 분석
- 암호화 PDF 분석
- 손상되거나 읽을 수 없는 PDF 분석
- DOCX, TXT, HWP, 이미지 파일의 공식 사용자 지원

TXT 파일은 개발 과정의 내부 테스트 입력으로 사용할 수 있으나, MVP의 공식 사용자 업로드 지원 형식으로 정의하지 않는다.

### 기술 확장

- Kubernetes
- 마이크로서비스 구조는 MVP 필수 범위에서 제외하며, 향후 서비스 규모와 독립 배포 필요성이 커질 경우 재검토할 수 있는 확장 후보로 둔다.
- 필수 Docker 구성
- 필수 Redis/Celery 구성
- 자체 AI 모델 운영
- 복잡한 검색 인덱스 또는 전문 검색 기능

Docker는 영구 제외 기술이 아니라 배포 재현성이나 개발 환경 통일이 필요해지는 후속 단계에서 검토할 수 있는 후보로 둔다.

## 8. 공식 파일 지원 범위

MVP의 공식 사용자 업로드 지원 형식은 텍스트 레이어가 있는 PDF로 한정한다.

업로드 파일이 MVP 지원 형식이 아닌 경우 사용자가 지원 불가 사실을 알 수 있어야 한다. 지원 불가 사유의 구체 문구, 오류 코드, 화면 위치는 후속 화면 및 API 설계에서 확정한다.

지원하지 않는 파일은 다음과 같다.

- 스캔 PDF
- 이미지 전용 PDF
- 암호화 PDF
- 손상되었거나 읽을 수 없는 PDF
- DOCX
- TXT
- HWP
- 이미지 파일
- OCR이 필요한 문서

파일 크기 제한, 페이지 수 제한, MIME 검증, 확장자 검증, 암호화 PDF 탐지, 손상 파일 탐지 방식은 후속 파일 처리 및 보안 설계 단계에서 확정한다.

## 9. 인증 최소 범위

MVP 인증은 이메일과 비밀번호 기반의 기본 계정 흐름으로 한정한다.

필수 범위는 다음과 같다.

- 이메일/비밀번호 회원가입
- 이메일/비밀번호 로그인
- 로그아웃
- 인증된 사용자의 본인 데이터 접근

MVP에서 제외하는 인증 기능은 다음과 같다.

- 소셜 로그인
- 이메일 인증
- 비밀번호 재설정
- 다중 역할
- 관리자 역할
- 기업 또는 조직 계정
- refresh token의 구체 정책 확정

토큰 저장 위치, access token 만료시간, refresh token 도입 여부, 로그아웃 및 강제 무효화 방식은 v0.2 개인정보/보안 설계와 v0.4 DB/API 설계에서 확정한다.

## 10. 분석 이력 최소 범위

MVP 분석 이력은 사용자가 자신의 분석 결과를 다시 확인할 수 있는 최소 수준으로 둔다.

필수 범위는 다음과 같다.

- 본인 분석 목록
- 문서 표시명
- 분석 생성일
- 분석 상태
- 분석 진행 중, 분석 완료, 분석 실패 상태 구분
- 완료된 분석 결과 재조회

제외 범위는 다음과 같다.

- 고급 검색
- 복잡한 필터
- 즐겨찾기
- 통계 대시보드
- 일괄 분석
- 관리자 조회

원본 파일명에 개인정보가 포함될 수 있으므로 문서 표시명 정책은 개인정보 및 화면 설계 단계에서 구체화한다.

## 11. 개인정보 보호 최소 요구사항

MVP는 개인정보 보호형 계약서 사전점검 서비스라는 포지션을 유지하기 위해 다음 방향을 필수 요구사항으로 둔다.

- 업로드 원본 파일을 영구 저장하지 않는다.
- 원본 계약서 파일 자체를 DB에 저장하지 않는다.
- 처리 완료 후 원본 파일을 삭제하는 방향을 적용한다.
- DB에는 문서 메타데이터, 분석 상태, 조항, 분석 결과만 저장하는 방향을 둔다.
- 계약서 원문, 마스킹 전 텍스트, 실제 사용자 요청/응답 원문, 검토 전 입력/출력 원문을 로그에 남기지 않는다.
- API Key, Secret, Token, Password를 저장소와 문서에 포함하지 않는다.

이번 단계에서는 개인정보 탐지 대상, 정규식, 마스킹 형식, 예외 규칙, 탐지 정확도, 오탐·미탐 처리, 구체 구현 방식을 확정하지 않는다.

정확한 삭제 시점, 삭제 실패 재시도, 실패 시 잔여 파일 처리, 임시 보관 시간, 정리 작업 방식, 감사 로그 범위는 v0.2 또는 파일 처리 설계 단계에서 확정한다.

## 12. AI 분석 최소 요구사항

MVP 분석 결과는 사용자가 계약서를 이해하고 추가 확인 질문을 준비할 수 있도록 다음 정보를 포함한다.

- 원문 조항 또는 연결 가능한 관련 문구
- 위험도 또는 검토 우선순위
- 위험 가능성 설명
- 해당 항목을 확인해야 하는 이유
- 추가로 확인할 질문
- 권장 행동
- 전문가 검토 권고 여부

이번 단계에서 확정하지 않는 사항은 다음과 같다.

- 위험도 단계명
- 점수 계산 방식
- confidence_score 기준
- AI 공급자
- AI 프롬프트
- JSON 응답 스키마
- 재시도 및 timeout 구체값
- 법적 효력 판단 기준

| 선행 기능 | 후행 기능 | 이유 |

| 파일 업로드 | 텍스트 추출 | 업로드된 문서가 분석 입력이 된다. |
| 텍스트 추출 | 개인정보 탐지 및 마스킹 | AI 분석 전에 개인정보 포함 가능 영역을 처리해야 한다. |
| 개인정보 탐지 및 마스킹 | 조항 분리 | 마스킹된 텍스트를 기준으로 분석 단위를 만들어야 한다. |
| 조항 분리 | AI 분석 | 조항별 위험 가능성을 분석해야 한다. |
| AI 분석 | 결과 조회 | 분석 결과가 있어야 사용자에게 보여줄 수 있다. |
| 원본 파일 삭제 정책 | 파일 처리 흐름 | 개인정보 보호 기준이 파일 처리와 함께 적용되어야 한다. |

## 14. MVP 완료 기준

MVP는 다음 조건을 만족하면 완료로 본다.

- 사용자가 가입, 로그인, 로그아웃을 할 수 있다.
- 인증된 사용자가 텍스트 레이어가 있는 PDF 계약서를 업로드할 수 있다.
- 지원하지 않는 파일을 업로드하면 사용자에게 지원 불가 사실을 안내할 수 있다.
- 서비스가 업로드 문서에서 텍스트를 추출하고 조항 단위로 나눌 수 있다.
- AI 분석 전에 개인정보 탐지 및 마스킹 처리가 수행된다.
- 마스킹 전 원문이 AI 분석 단계로 직접 전달되지 않는 방향을 만족한다.
- 조항별 위험 가능성과 확인 필요 사항을 분석할 수 있다.
- 분석 결과에 관련 문구, 위험 가능성 설명, 확인 이유, 추가 질문, 권장 행동, 전문가 검토 권고 여부가 포함된다.
- 사용자가 분석 진행 중, 완료, 실패 상태를 구분할 수 있다.
- 분석 실패 시 실패 여부를 확인할 수 있다.
- 사용자가 본인의 분석 목록과 완료된 분석 결과를 다시 확인할 수 있다.
- 업로드 원본 파일을 영구 저장하지 않는 방향이 구현되어 있다.
- 계약서 원문, 개인정보, Secret, 검토 전 입력/출력 원문이 공개 저장소와 로그에 포함되지 않는다.
- 서비스가 법률 자문이나 전문가 검토 대체가 아니라 참고용 사전점검 도구임을 명확히 설명한다.

## 15. 포트폴리오 시연 범위

포트폴리오 시연에서는 다음 흐름을 보여주는 데 집중한다.

- 사용자가 로그인한다.
- 텍스트 레이어가 있는 PDF 계약서를 업로드한다.
- 분석 진행 상태를 확인한다.
- 조항별 분석 결과를 확인한다.
- 전문가에게 질문할 항목을 정리한다.
- 분석 이력에서 완료된 결과를 다시 연다.

시연에는 실제 개인정보나 실제 계약서 원본을 사용하지 않고, 공개 가능한 가상 샘플만 사용한다.
