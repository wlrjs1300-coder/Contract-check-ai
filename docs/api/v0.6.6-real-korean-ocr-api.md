# v0.6.6 실전 한글 OCR API 정합성

## 1. 적용 범위

- `POST /extractions/images`의 실제 로컬 OCR 경로를 `OCR_ADAPTER=local_korean`(또는 `local`, `tesseract`)로 활성화한다.
- `APP_ENV=test` 기본 동작은 기존처럼 synthetic adapter를 사용한다.
- non-test 기본 동작은 안전 실패 `ocr_unavailable`(fail-closed)이다.
- 외부 OCR API/Provider는 사용하지 않는다.

## 2. 실측 경로

- 요청 검증·전처리: `POST /extractions/images` 동일.
- OCR 실행: `LocalKoreanOcrAdapter` → 정규화 PNG의 픽셀 입력에서 `image_to_data` 기반 블록 추출.
- 출력 매핑: 텍스트, blocks(text/confidence/bbox/reading_order), page confidence.
- 성공 결과는 항상 `review_required=true`, `analysis_blocked=true`로 저장/조회된다.
- `extraction status`는 사용자 확인/수정 이전에는 분석 파이프라인으로 전이하지 않는다.

## 3. 오류 코드(핵심)

- `ocr_unavailable`: non-test에서 OCR adapter 미활성 또는 경로 차단
- `ocr_engine_unavailable`: 로컬 실행 엔진(binary) 미설치
- `ocr_model_unavailable`: 언어팩(예: kor, eng) 로드 실패
- `ocr_failed`: adapter 처리 예외
- `ocr_result_invalid`: OCR 결과 스키마 불일치
- `empty_ocr_result`: 추출 텍스트 없음

## 4. 테스트 반영

- 기존 이미지 단위 계약 테스트는 synthetic adapter override로 유지.
- 실 엔진 경로는 설정 환경에서만 진입하도록 분기 테스트 추가.
- 통합 테스트는 현재 실행 가능한 파이프라인에서 synthetic/mocked 경로로 확정되며, 실제 엔진 검증은 설치 가능한 환경에서 별도 확장 스텝으로 수행한다.
- v0.6.7 사진 품질 강화 단계에서 품질 코드(`image_blurry`, `image_quality_too_low`, `deskew_uncertain` 등)를 경고군으로 보완하고, 경고가 실제 OCR 문자열(`ocr_amount_format_suspected`)과 결합되는지 회귀 검사한다.

## 4. 운영/보안 경계

- 원본·파생 파일은 기존 `RequestDirectory` 정리 경로를 따르며 cleanup 실패 시 500으로 차단.
- `ocr_*` 내부 엔진/모델 경로, 절대 경로, 내부 예외 메시지를 사용자 응답에 노출하지 않는다.
- 외부 전송 없음으로 개인정보 원문이 제3자 경계로 이동하지 않는다.
