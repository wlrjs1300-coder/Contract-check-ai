## Provider 스키마

요청 schema_version:

- `analysis-provider-request.v1`

응답 schema_version:

- `analysis-provider-response.v1`

요구사항:

- `schema_version`이 누락되면 거부 (`provider_schema_version_missing`)
- 지원되지 않는 버전은 거부 (`provider_schema_version_unsupported`)
- 요청의 `request_id`가 비어있으면 거부 (`provider_request_id_missing`)
- 응답의 `provider_request_id`가 요청과 일치하지 않으면 거부 (`provider_request_id_mismatch`)
- 요청/응답 바이트 한계는 UTF-8 바이트 기준으로 사전 검사
- `provider_request_too_large`, `provider_response_too_large`는 요청/응답 생성 직후 저장되지 않음

## 인터페이스

Provider는 외부 호출 없이 결정론적으로 동작할 수 있어야 하며, 현재 v0.7 구현 범위는 다음 인터페이스만 정의한다.

- `analyze(request: AnalysisProviderRequest) -> AnalysisProviderResponse`
- 실패 응답은 계약 코드로 변환되어 반환
- 예시 오류 코드:
  - `provider_schema_version_missing`
  - `provider_schema_version_unsupported`
  - `provider_request_id_mismatch`
  - `provider_request_too_large`
  - `provider_response_too_large`
  - `provider_finding_limit_exceeded`
  - `provider_candidate_mismatch`
  - `provider_evidence_missing`

## 실패/재시도 계약

- timeout, rate limit, temporary failure는 retry 대상
- permanent failure, schema failure, request/response size failure, unknown candidate는 retry 불가
- retry 초과 시 `provider_retry_exhausted`로 종료
- attempt 수와 request_id는 추적되며 내부 request payload는 저장/노출 대상 아님

## v0.7.1 PII Gate 확장

- Provider 요청/응답 경계에서 추가 실패 코드
  - `provider_data_minimization_failed`
  - `provider_redaction_token_collision`
  - `provider_output_pii_detected`
- 요청 본문은 `masked` clause 텍스트와 최소 evidence 정보만 포함한다.
- `document_id`는 공개 노출 가능한 원문 ID 대신 불투명 식별값으로 전송한다.
- 파일명·경로·raw clause 본문은 request payload에 직접 포함되지 않는다.
- Provider 출력은 출력 안전성 및 출력 PII 스캔을 모두 통과해야 한다.

## raw payload 및 로그 경계

- raw request/response payload 저장 금지
- clause 전체 텍스트, evidence source text, 파일 경로, provider 메시지의 내부 예외 상세를 로그/DB/API에 노출하지 않음
- 사용자는 `failed`, `failed_detail_code`, `retryable` 같은 안정적 표식만 확인

## 사용량(usage) 계약

- usage 필드는 음수 금지
  - input_units
  - output_units
  - total_units
  - estimated_cost_minor
  - currency
  - provider_reported
- 과도한 usage 값은 정량 검사로 거부
- 모델 가격 정책(정적 금액)은 사용하지 않음

## v0.7.0 범위 (현재)

- 실제 외부 Provider 연동은 미구현
- provider contract 검증, factory 정책, 실행/재시도, 보안 경계, 테스트 계약을 선행으로 고정
- 결과의 정확성/법률 판단은 실제 AI Provider 연동과 별도 단계에서 보강
