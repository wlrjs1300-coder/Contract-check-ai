# v0.6.8 페이지 근거·조항 연결 API 계약 (요약)

## 스냅샷 해시/버전

- `confirmation_snapshot`의 해시는 문자열 정규화(UTF-8, LF, NFC) 후 canonical JSON 기반 SHA-256로 산출한다.
- 분석 응답에는 `snapshot_version` 및 `snapshot_stale`을 반환한다.

## 분석 결과 응답

`GET /documents/{document_id}/analysis-results`는 기존 결과 항목에 아래 항목을 함께 반환한다.

- `finding_id`
- `severity`
- `title`
- `recommendation`
- `is_stale`
- `evidence` (목록)

각 evidence 항목은 다음을 포함한다.

- `evidence_id`
- `page_number`
- `clause_id`
- `source_text`
- `start_offset` / `end_offset` (문자 단위, inclusive/exclusive)
- `block_ids`
- `evidence_type`
- `validation_status`
- `snapshot_version`

## 증거 검증 코드

- `evidence_reference_invalid`
- `evidence_snapshot_mismatch`
- `evidence_page_not_found`
- `evidence_block_not_found`
- `evidence_clause_not_found`
- `evidence_offset_invalid`
- `evidence_text_mismatch`
- `evidence_ambiguous`
- `evidence_missing`

## 제약

- provider raw payload, 파일 경로, 내부 전체 텍스트는 API로 노출하지 않는다.
- 추출 fixture는 합성 데이터 사용.
