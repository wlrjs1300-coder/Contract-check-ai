# v0.7.5 저장 암호화·키 관리 로드맵

## 목표

v0.7.5는 저장 암호화, keyring, key rotation 기반, fail-closed 오류 계약, migration과 테스트를 실제 구현한다. 현재 문서 집합은 구현 전에 정책과 경계를 확정한다. v0.7.5 완료는 보관형 실운영이나 실제 개인정보 사용의 자동 승인을 의미하지 않는다.

## 1. 설계 확정

- A/B/C/D 필드 분류 확정
- 인증 주체와 부모 기반 owner 경계 확인
- envelope, canonical AAD, 오류 계약 확정
- filename·nested JSON·migration 미확정 항목 결정

완료 기준:

- ADR 상태와 구현 전 결정 항목 일치
- 문서 간 평문 fallback, AAD, key status 정책 일치

## 2. Crypto core 및 keyring

- keyring parsing과 strict schema 검증
- AES-256-GCM encrypt/decrypt
- Base64 strict encode/decode
- canonical AAD 생성
- 안전한 오류 코드 정규화
- key status와 active key 검증

완료 기준:

- malformed·누락·중복 keyring fail-closed
- raw key 비노출
- envelope 누락·추가·tamper 거부

## 3. Scalar 필드 암호화

- encrypted backing column 검토 및 구현
- `Clause.body`, `ExtractionPage.text`, `AnalysisResultItem.summary` 우선 적용
- B등급 filename·title·구조화 값의 API 호환성 결정
- 명시 helper를 통한 암복호화

완료 기준:

- raw DB에서 대상 평문 부재
- 권한 확인 뒤 API 응답 직전에만 명시 복호화
- ORM 객체에 평문 장기 보관 금지

## 4. Nested JSON 암호화

- 모델별·schema별 allowlist와 validator
- snapshot, reviewed/final text, blocks text 암호화
- evidence와 분석 자연어 하위 필드 암호화
- 평문 운영 메타데이터 분리
- 저장 금지 payload 차단

완료 기준:

- 전체 JSON 투명 암호화 없음
- 이중 암호화와 임의 민감 key 저장 차단

## 5. Migration

- 개발 DB 실제 개인정보 부재 확인
- 기본 전략인 개발 DB 초기화 후 encrypted schema 전환
- 필요 시 합성 데이터 전용 one-time re-encryption
- production 평문 fallback과 장기 dual-write 차단
- upgrade·downgrade·부분 실패·재실행 검증

완료 기준:

- 민감 row 발견 시 fail-closed
- plaintext·encrypted 동시 저장 거부
- migration 후 plaintext column 제거 계획 확인

## 6. API·서비스 통합

- owner context와 AAD 재구성
- 복호화 전 소유권 검증
- API serializer 직전 명시 복호화
- ciphertext·nonce·내부 key 정보 비노출
- cross-user 일반 `404` 유지

완료 기준:

- 문서·추출·분석 경로 회귀 통과
- 복호화 실패 시 평문 fallback 없음

## 7. Rotation 기반

- active·decrypt_only·retired 상태 구현
- active key 신규 쓰기
- 기존 `key_id` 복호화
- key usage inspection
- 한 건 또는 제한된 batch 재암호화 helper
- 재암호화 idempotency 검증

완료 기준:

- decrypt_only 신규 encrypt 거부
- retired 전 사용 row 0건과 backup 영향 확인
- 신규 key 전환 뒤 기존·신규 ciphertext 동시 읽기 검증

후속 범위:

- background bulk re-encryption worker
- KMS 자동 rotation
- multi-region rotation
- backup 전체 재암호화
- 운영 대시보드 orchestration

## 8. 독립 테스트와 문서 완료

- keyring·envelope·AAD·저장 경계·rotation 테스트
- raw DB 평문 부재 검사
- 로그·예외·응답 비노출 검사
- 전체 backend 회귀와 정적 검사
- 구현과 공개 문서 정합성 검토

최종 완료 조건:

- 코드·migration·테스트가 모두 구현됨
- fail-closed와 key rotation 기반이 검증됨
- 실제 개인정보 파일럿은 별도 보안 게이트 전까지 금지됨
