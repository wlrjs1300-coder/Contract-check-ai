# 버전 관리 규칙

## 1. 목적

ContractCheck AI 프로젝트는 작업을 버전 단위로 나누어 관리한다.

각 버전은 하나의 명확한 목적, 작업 범위, 검증 기록, 릴리스 기록을 가진다.

## 2. 버전 형식

```text
v<major>.<minor>.<patch>
```

예시:

```text
v0.0.1
v0.1.3
v0.2.5
v1.0.4
```

## 3. 버전 의미

| 구분 | 의미 |
|---|---|
| major | 큰 릴리스 단위. MVP 구현 완료 시 v1.0 사용 |
| minor | 큰 작업 영역 단위 |
| patch | 세부 작업 단위 |

## 4. 큰 흐름

```text
v0.0 프로젝트 운영 준비
v0.1 서비스 기획
v0.2 개인정보/보안 설계
v0.3 화면/UX 설계
v0.4 DB/API 설계
v0.5 분석 파이프라인 설계
v0.6 개발 환경 세팅
v1.0 MVP 구현
```

## 13. 변경 가능성

본 버전 계획은 기준표이며, 개발 과정에서 변경될 수 있다.

변경 시 다음 내용을 체크리스트에 기록한다.

- 변경 전 계획
- 변경 후 계획
- 변경 사유
- 영향 범위
- 다음 작업
