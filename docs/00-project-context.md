# ContractCheck AI 프로젝트 기본 컨텍스트

## 1. 프로젝트명

ContractCheck AI

## 2. 부제

개인정보 보호형 계약서 사전점검 서비스

## 3. 프로젝트 목적

ContractCheck AI는 사용자가 업로드한 계약서를 분석하여 위험 조항, 모호한 표현, 검토 필요 항목을 사전에 확인할 수 있도록 돕는 포트폴리오용 AI 문서 분석 서비스이다.

본 프로젝트는 단순 CRUD 서비스가 아니라, 계약서 업로드부터 텍스트 추출, 개인정보 마스킹, 조항 분할, AI 리스크 분석, 결과 대시보드까지 이어지는 분석 파이프라인 구현을 목표로 한다.

## 4. 서비스 포지션

ContractCheck AI는 법률 자문 서비스가 아니다.

본 서비스는 계약서 체결 전 사용자가 놓치기 쉬운 위험 조항과 모호한 표현을 확인할 수 있도록 돕는 참고용 사전점검 도구이다.

## 5. MVP 지원 계약서

전체 공식 지원 계획은 다음 두 계약서를 유지한다.

1. 근로계약서
2. 주택 임대차계약서

v1.0 최초 구현과 기술 검증은 근로계약서 1종을 우선한다. 주택 임대차계약서는 전체 지원 계획에는 유지하되 후속 구현 대상으로 둔다. 상세 근거와 화면 노출 방향은 `docs/planning/user-scenarios.md`를 따른다.

## 6. 확장 후보

MVP 이후 다음 계약서로 확장할 수 있다.

1. 상가 임대차계약서
2. 프리랜서/용역계약서
3. 비밀유지계약서

## 7. 핵심 MVP 기능

- 회원가입/로그인
- 계약서 업로드
- 텍스트 추출
- 개인정보 탐지 및 마스킹
- 조항 자동 분할
- 조항별 리스크 분석
- 확인 우선순위별 결과 대시보드
- 조항 상세 분석 화면
- 분석 이력 관리
- 원본 파일 자동 삭제 정책

## 8. 기술스택 방향

- Frontend: React + Vite
- UI: Bootstrap 5 + 직접 작성 CSS
- Backend: FastAPI
- Database: MySQL
- Auth: JWT
- AI API: 제공자 교체가 가능하도록 추상화
- Docs: Markdown
- Git: GitHub + branch/commit/PR/tag 규칙 기반 진행

## 9. 핵심 설계 원칙

- 원본 계약서를 외부 분석 API에 직접 전달하지 않는다.
- 개인정보 마스킹 후 분석을 수행한다.
- 원본 파일은 분석 완료 후 삭제한다.
- DB에는 마스킹된 조항과 분석 결과 중심으로 저장한다.
- 로그에 계약서 원문, 개인정보, API Key가 남지 않도록 한다.
- 분석 결과는 참고용이며 최종 법률 판단으로 표현하지 않는다.
- 분석 결과는 점수, 퍼센트, 확률 기반 위험도로 표시하지 않는다. `핵심 확인`, `추가 확인`, `조건 확인`, `전문가 검토 권고`는 이를 행동 중심 비수치 방식으로 표현하기 위한 후보 체계이며, 최종 단계명과 표시 방식은 `docs/planning/mvp-scope.md`, `docs/planning/service-responsibility-policy.md` 및 v0.3·v0.5 설계 단계에서 확정한다.

## 10. 운영 원칙

- 버전 계획은 기준표로 사용하되, 진행 중 변경될 수 있다.
- 변경이 발생하면 기존 계획을 삭제하지 않고 변경 사유를 체크리스트에 기록한다.
- 모든 작업은 버전별 체크리스트 MD 파일을 기준으로 진행한다.
- 공개 저장소에는 정제된 산출물만 포함한다.
