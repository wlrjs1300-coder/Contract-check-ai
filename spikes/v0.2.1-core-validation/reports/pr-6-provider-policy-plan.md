# PR-6 AI 공급자 정책 평가 계획

## 1. 문서 목적

이 문서는 PR-2~PR-5의 기술 관찰 결과를 바탕으로 외부 AI 공급자 사용 가능 조건을 검토하기 위한 조사 계획 문서다.

- 특정 공급자 채택을 미리 결정하지 않는다.
- 실제 API 호출이나 데이터 전송 없이 공식 공개 문서만 검토한다.
- 실제 계약서, 개인정보, 민감정보를 사용하지 않는다.
- 이 문서는 법률 자문이나 개인정보 보호 적합성 인증이 아니다.
- 최종 판단은 MVP 범위 유지, 축소, 보완 여부를 결정하기 위한 기술 의사결정 보조 목적으로만 사용한다.

## 4. 공식 출처 원칙

조사에는 다음 출처만 사용한다.

- 공급자 공식 API 문서
- 공급자 공식 개인정보 처리방침
- 공급자 공식 데이터 사용·보관 정책
- 공급자 공식 보안·컴플라이언스 문서
- 공급자 공식 가격 정책
- 공급자 공식 서비스 약관
- 공급자 공식 변경 공지, 도움말 또는 릴리스 노트

보조적으로만 사용할 수 있는 출처:

- 정부기관 공식 자료
- 규제기관 공식 자료
- 국제 표준기관 공식 자료

사용하지 않는 출처:

- 블로그 요약
- 광고성 비교 글
- 커뮤니티 게시물
- 검색 결과 요약문
- 출처 없는 2차 자료
- 공식 문서로 확인하지 않은 추정
- 일반 소비자용 챗봇 서비스 정책을 API 정책과 혼용한 자료

각 근거에는 다음 메타데이터를 기록한다.

- 공급자
- 문서 제목
- 공식 URL
- 확인일
- 문서 갱신일 또는 시행일
- 적용 제품 또는 API 범위
- 근거 요약
- 불확실성 또는 추가 확인 필요 여부

## 5. 필수 비교 항목

### 5.1 입력 데이터의 학습 사용

- API 입력과 출력이 기본적으로 모델 학습에 사용되는지 확인한다.
- opt-in 또는 opt-out 구조를 확인한다.
- 별도 계약이나 제품 등급에 따라 달라지는지 확인한다.

### 5.2 데이터 보관

- 입력과 출력의 기본 보관 기간을 확인한다.
- abuse monitoring 또는 safety monitoring 목적 보관 여부를 확인한다.
- 보관 기간 축소 또는 zero-retention 옵션 존재 여부를 확인한다.
- 옵션 사용 조건과 제한을 확인한다.

### 5.3 사람의 접근 가능성

- 공급자 인력 또는 검토자가 데이터에 접근할 수 있는 조건을 확인한다.
- 남용 탐지, 고객 지원, 법적 의무 등 예외 조건을 확인한다.
- 접근 통제와 기록 관련 공개 근거를 확인한다.

### 5.4 지역과 국외 이전

- 데이터 처리 지역을 확인한다.
- 데이터 거주지 또는 지역 고정 옵션을 확인한다.
- 대한민국 사용자 데이터의 국외 이전 가능성을 확인한다.
- 하위 처리자 또는 subprocessors 공개 여부를 확인한다.

### 5.5 삭제와 보존 종료

- 고객 요청에 따른 삭제 가능 여부를 확인한다.
- 계정 종료 시 데이터 처리 방식을 확인한다.
- 백업, 로그, 법적 보존 예외를 확인한다.

### 5.6 보안과 컴플라이언스

- 전송 및 저장 암호화 공개 근거를 확인한다.
- 접근 통제 공개 근거를 확인한다.
- SOC 2, ISO 27001 등 공개 인증 보유 여부를 확인한다.
- DPA 또는 기업용 계약 제공 여부를 확인한다.
- 인증 보유가 프로젝트 적합성을 자동 보장하지 않는다는 점을 기록한다.

### 5.7 API 기능 적합성

- 구조화 출력 또는 JSON 응답 지원 여부를 확인한다.
- 긴 문서 처리 가능성을 확인한다.
- 모델별 context window를 확인한다.
- 출력 길이 제한을 확인한다.
- 재현성과 temperature 제어 가능성을 확인한다.
- 안전 필터 또는 거절 동작이 계약서 분석에 미치는 영향을 검토한다.

### 5.8 가격과 운영 제약

- 입력·출력 토큰 가격 구조를 확인한다.
- 무료 사용량 또는 체험 정책을 확인한다.
- rate limit을 확인한다.
- 비용 상한 통제 가능성을 확인한다.
- 모델·가격 변경 위험을 기록한다.
- 가격은 조사 시점 기준이며 고정값이 아님을 명시한다.

### 5.9 계약서 분석 프로젝트 적합성

- 마스킹된 계약서만 전송하는 구조에 적합한지 검토한다.
- 원문 전송 금지를 기술적으로 강제할 수 있는지 검토한다.
- raw request/response 저장 금지 정책을 구현할 수 있는지 검토한다.
- reference_id 기반 결과 검증 구조와 결합 가능한지 검토한다.
- 공급자 교체 가능한 adapter 경계가 필요한지 검토한다.

## 6. 판정 등급

공급자별 항목 판정은 다음 4단계로 고정한다.

- PASS: 공식 근거가 명확하고 MVP 안전 조건을 충족할 가능성이 높음
- CONDITIONAL: 별도 설정, 계약, 제품 등급, 보관 제한 또는 운영 통제가 있어야 사용 가능
- REVIEW: 공식 근거가 불충분하거나 정책·제품별 차이가 있어 추가 확인 필요
- BLOCK: 현재 MVP 안전 원칙과 충돌하거나 필요한 통제가 제공되지 않음

주의 사항:

- 공급자 전체를 단순 점수 하나로 평가하지 않는다.
- 법률 적합성을 수치화하지 않는다.
- 인증 개수만으로 공급자를 선정하지 않는다.
- 공개되지 않은 내용은 추정하지 않고 REVIEW로 분류한다.
- zero-retention이라는 용어만으로 완전 무보관을 단정하지 않는다.

## 7. MVP 최소 안전 조건

최종 의사결정에서 다음 조건을 필수 게이트로 검토한다.

- 실제 계약서 원문 외부 전송 금지
- PR-3 마스킹 처리 후 데이터만 전송
- 마스킹 실패 또는 residual 발견 시 외부 전송 BLOCK
- 전송 전 허용 필드 allowlist 적용
- raw input 저장 금지
- raw external response 저장 금지
- 최소 로그만 저장
- 로그에 계약서 본문이나 개인정보 포함 금지
- 외부 AI 출력에 PR-5 출력 안전성 검증 적용
- reference_id 검증 실패 시 결과 BLOCK 또는 REVIEW
- 공급자 정책과 모델 버전 기록
- 비용 제한 설정
- 네트워크 timeout과 재시도 제한
- 공급자 장애 시 안전 실패 처리
- 공급자 교체 가능한 adapter 경계 필요
- 실제 개인정보와 실제 계약서를 사용하는 테스트는 별도 승인 없이는 금지
